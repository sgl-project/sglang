COMPARISON: FP8 vs BF16 PATH
─────────────────────────────────────────────────────────────────────────────

FP8 PATH (Quantized):
  Input (BF16) → Quantize → FP8 → DeepGEMM Kernel → BF16 Output
  Overhead: 5-10% (quantization)
  Speedup: 50-70% (GEMM)
  Net: 20-30% faster
  Accuracy: ~0.1-0.5% loss

BF16 PATH (No Quantization):
  Input (BF16) → F.linear(x, w, b) → BF16 Output
  No overhead, no speedup
  Full accuracy

QUANTIZATION FORMULA (Per-Token-Group)
─────────────────────────────────────────────────────────────────────────────

For each group of elements:
  1. Calculate: scale = max(abs(values)) / fp8_max
  2. Quantize: quantized_value = clamp(value / scale, fp8_min, fp8_max)
  3. Store: both quantized_value (FP8) and scale (Float32 or UE8M0)

DeepGEMM Computation:
  1. Dequantize activations: A_float = A_fp8 * A_scales
  2. Dequantize weights: B_float = B_fp8 * B_scales
  3. GEMM in BF16: C = A_float @ B_float^T
  4. Store result in BF16
