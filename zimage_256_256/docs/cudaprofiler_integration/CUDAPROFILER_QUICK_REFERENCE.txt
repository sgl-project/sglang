┌─ HOW IT WORKS WITH NSYS ────────────────────────────────────────────────────────┐
│                                                                                 │
│  STEP 1: Launch with nsys wrapper                                              │
│          $ nsys profile --capture-range=cudaProfilerApi -o trace.nsys-rep \    │
│            python -m sglang.bench_one_batch --profile-activities CUDA_PROFILER │
│                                                                                 │
│  STEP 2: nsys starts background tracing                                        │
│          ↓ Waits for application to call cudaProfilerStart()                  │
│                                                                                 │
│  STEP 3: Application execution                                                 │
│          ├─ Load model (NOT captured)                                          │
│          ├─ Warmup inference (NOT captured)                                    │
│          ├─ Actual inference request:                                          │
│          │  ├─ Call cudaProfilerStart()  ← nsys STARTS capturing              │
│          │  ├─ Execute forward pass       ← All GPU ops recorded              │
│          │  ├─ torch.cuda.synchronize()  ← Wait for GPU to finish             │
│          │  └─ Call cudaProfilerStop()   ← nsys STOPS capturing               │
│          └─ Return results                                                     │
│                                                                                 │
│  STEP 4: nsys generates trace file                                             │
│          trace.nsys-rep  ← Can be analyzed with nsys GUI/CLI                   │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘

┌─ KEY FEATURES ──────────────────────────────────────────────────────────────────┐
│                                                                                 │
│  ✓ AUTOMATIC WARMUP EXCLUSION                                                 │
│    → Scheduler checks req.is_warmup flag                                       │
│    → Only non-warmup requests trigger profiler                                 │
│    → Warmup latency doesn't pollute profiling data                             │
│                                                                                 │
│  ✓ MINIMAL OVERHEAD                                                            │
│    → Just 2 API calls: cudaProfilerStart() and cudaProfilerStop()             │
│    → No torch.profiler tracing overhead                                        │
│    → Suitable for production profiling                                         │
│                                                                                 │
│  ✓ MULTI-GPU SAFETY                                                            │
│    → Only base_gpu_id issues profiler commands                                 │
│    → Prevents trace file conflicts                                             │
│    → Works in distributed/multi-node settings                                  │
│                                                                                 │
│  ✓ SYNCHRONIZATION                                                             │
│    → torch.cuda.synchronize() called before cudaProfilerStop()               │
│    → Ensures all GPU work is complete before ending capture                    │
│    → Prevents incomplete traces                                                │
│                                                                                 │
│  ✓ STAGE-BASED PROFILING                                                      │
│    → Can profile specific stages (prefill/decode)                              │
│    → ProfileManager supports stage-based triggering                            │
│    → Useful for identifying bottleneck phases                                  │
│                                                                                 │
└─────────────────────────────────────────────────────────────────────────────────┘

┌─ PROFILE ACTIVITIES (Activity Types) ────────────────────────────────────────────┐
│                                                                                 │
│  CPU         → CPU profiling via torch.profiler                                │
│  GPU         → GPU profiling via torch.profiler (CUDA)                         │
│  CUDA_PROFILER → CUDA profiler (nsys compatible)                               │
│  MEM         → Memory profiling (torch.cuda.memory._record_memory_history)    │
│  RPD         → ROCM profiler (for AMD GPUs)                                    │
│  XPU         → Intel XPU profiling                                             │
│                                                                                 │
│  NOTE: CUDA_PROFILER is the ONLY activity that uses cudaProfilerStart/Stop   │
│        Other activities use torch.profiler internally                          │
│                                                                                 │

┌─ COMPARISON: cudaProfilerStart vs torch.profiler ──────────────────────────────┐
│                                                                                 │
│  Feature                   | cudaProfilerStart | torch.profiler                │
│  ───────────────────────── | ──────────────── | ──────────────                 │
│  Requires external tool    | YES (nsys)       | NO (standalone)                │
│  API call overhead         | Minimal (2)      | High (tracing all ops)        │
│  Captures framework init   | NO (controlled)  | YES (always)                   │
│  Warmup exclusion          | Manual           | Via schedule()                 │
│  Multi-GPU safety          | Per rank guard   | Works on all ranks             │
│  Output format             | nsys-rep         | JSON + gzip                    │
│  Trace size                | Small (range)    | Large (full trace)             │
│  Best for                  | Production       | Development                    │
│                                                                                 │

┌─ TROUBLESHOOTING ───────────────────────────────────────────────────────────────┐
│                                                                                 │
│  Issue: "Failed to start CUDA profiler"                                       │
│  → Check: nsys is installed (nsys --version)                                   │
│  → Check: Running with --capture-range=cudaProfilerApi flag                    │
│  → Fix: nsys profile --capture-range=cudaProfilerApi python ...               │
│                                                                                 │
│  Issue: Multiple trace files on multi-GPU                                      │
│  → Cause: All GPUs calling cudaProfilerStart()                                 │
│  → Fix: Scheduler only calls on base_gpu_id (already handled)                  │
│  → Verify: Check base_gpu_id in server_args                                    │
│                                                                                 │
│  Issue: Trace file is empty or corrupted                                       │
│  → Cause: GPU operations not complete before cudaProfilerStop()              │
│  → Fix: torch.cuda.synchronize() is called automatically                       │
│  → Manual fix: Ensure proper GPU sync before stopping                          │
│                                                                                 │
│  Issue: nsys not capturing GPU operations                                      │
│  → Cause: Application not calling cudaProfilerStart()                          │
│  → Check: Verify --profile-activities CUDA_PROFILER is set                     │
│  → Verify: Requests are not all marked as warmup                               │
│                                                                                 │
