# nsys artifacts (conc=32/128K/tb=8192 winning point)

Heavy artifacts (.nsys-rep + .sqlite) live outside the repo at
`/workspace/nsys_reports/`:

  ds_native_off_131072_c32.nsys-rep   (372 MB)
  ds_native_off_131072_c32.sqlite     (935 MB — generated by `nsys stats`)
  ds_native_on_131072_c32.nsys-rep    (1.06 GB)
  ds_native_on_131072_c32.sqlite      (2.6 GB — generated by `nsys stats`)

In-repo artifacts (this directory):

  off_c32_nsys.json    DS-off bench-decode JSON (output_len=64, no NIAH)
  on_c32_nsys.json     DS-on bench-decode JSON (tb=8192, output_len=64)
  compare_c32.txt      Output of `compare_nsys.py` — the kernel diff

To regenerate the diff against fresh nsys-rep files:

  PYTHONPATH=python python3 benchmark/double_sparsity/repro_session/compare_nsys.py \
      /workspace/nsys_reports/ds_native_off_131072_c32.nsys-rep \
      /workspace/nsys_reports/ds_native_on_131072_c32.nsys-rep \
      > compare_c32.txt

To regenerate the .nsys-rep files (~15 min on 8x H200):

  nsys profile --trace cuda,nvtx --force-overwrite=true --stats=false \
      -o /workspace/nsys_reports/ds_native_off_131072_c32 \
      -- env PYTHONPATH=python SGLANG_ALLOW_OVERWRITE_LONGER_CONTEXT_LEN=1 \
        python3 benchmark/double_sparsity/bench_decode.py \
        --config branch_ds_off --model meta-llama/Llama-3.1-70B-Instruct \
        --context-len 131072 --output-len 64 --n-requests 32 --concurrency 32 \
        --tp-size 8 --mem-fraction-static 0.85 --max-running-requests 32 \
        --output-json /workspace/nsys_reports/off_c32_nsys.json

  # ... and the same with --config branch_ds_on + DS args (see SESSION_REPORT §10
  # reproduction recipe) and a different -o path.
