# yc_self_learn 文件命名规则

## 📋 概述

本文档定义了 `yc_self_learn` 目录下所有文件的命名规范和组织规则，确保文档结构清晰、易于导航和管理。

---

## 🎯 核心原则

1. **层级清晰**：通过前缀和编号体现文档的层级关系
2. **分类明确**：通过字母/数字组合区分不同类型的文档
3. **易于查找**：命名规范统一，便于快速定位
4. **可扩展性**：支持子文档、多层级结构

---

## 📁 目录结构与命名规则

### 1. `md/` 目录 - 纯数字编号系统

**用途**：存放 SGLang 学习相关的核心文档

**命名格式**：
```
[数字]_[描述].md
[数字]_[子编号]_[描述].md
```

**规则**：
- 使用**纯数字**作为主编号，按创建顺序递增：`03_`, `04_`, `06_`, `07_`, `08_`, `09_`, `10_`, `11_`, `12_`, `13_`, `14_`, `15_`...
- 子文档使用 `[主编号]_[子编号]_` 格式：`13_01_`, `13_02_`, `13_03_`
- 描述部分使用**下划线**分隔，可以是中文或英文
- 描述通常包含"详解"、"指南"等后缀

**示例**：
```
03_TTFT_为什么重要.md
06_RadixAttention详解.md
07_多请求场景与批处理.md
08_process_batch具体实现详解.md
13_01_SGLang_学习指南.md
13_02_SGLang_开发重点与技术创新.md
13_03_SGLang_Scheduler_技术变迁.md
14_Cache_Aware_Router前缀匹配详解.md
22_SGLang完整请求流程详解_纠正版.md
34_Issue_17031_详细分析_高并发流式请求错误.md
```

**特点**：
- ✅ 简单直观，按时间顺序编号
- ✅ 支持子文档扩展（`13_01_`, `13_02_`）
- ✅ 适合线性学习路径

### 2. `llm study sglang_yc01252026/` 目录 - Z/Y 系列 + 日期系列

**用途**：SGLang 系统化学习文档，使用更结构化的命名

**命名格式**：
```
[层级前缀]_[主题代码]_[描述].md
[层级前缀]_Day[数字]_[主题代码]_[描述].md
```

#### 2.1 层级前缀规则

| 前缀 | 用途 | 示例 |
|------|------|------|
| `00_` | 主文档/索引文档/基础概念 | `00_基础概念完整学习指南.md`<br>`00_Z7_GPU基本计算单元_SM_详解.md` |
| `01_` | 第一类子文档/官方文档索引 | `01_官方文档快速索引.md`<br>`01_Z3_量化技术对比_详解.md` |
| `02_` | 学习计划类文档 | `02_学习计划.md` |
| `03_` | Issue 相关文档 | `03_Issue_17526_学习路径.md`<br>`03_Z1_为什么需要理解CUDA和Kernel开发_详解.md` |
| `DAY04_` | 特定日期的文档 | `DAY04_Z1_SGLang_CI_CD系统详解.md` |

#### 2.2 Z 系列主题分类

**格式**：`00_Z[数字]_[描述]_详解.md`

**规则**：
- Z 系列表示**主题分类**，按主题编号：`Z1_`, `Z2_`, `Z3_`, `Z4_`, `Z5_`, `Z6_`, `Z7_`, `Z8_`
- 每个 Z 主题可以有自己的子文档（Y 系列）

**示例**：
```
00_Z1_Scaled_Dot_Product_Attention_详解.md
00_Z2_梯度消失和梯度爆炸_详解.md
00_Z3_Attention_Head_详解.md
00_Z4_Multi_Head_如何学习不同关注模式_详解.md
00_Z5_如何验证Head的关注模式_详解.md
00_Z6_Blackwell_GPU_架构_详解.md
00_Z7_GPU基本计算单元_SM_详解.md
00_Z8_Feed_Forward_Network_详解.md
```

#### 2.3 Y 系列子文档

**格式**：`[层级前缀]_Z[数字]_Y[数字]_[描述]_详解.md`

**规则**：
- Y 系列表示**Z 主题的子文档**
- 格式：`00_Z07_Y01_`, `00_Z07_Y02_`, `01_Z3_Y01_`, `01_Z3_Y02_`
- Y 编号从 `Y01` 开始递增

**示例**：
```
00_Z07_Y01_Thread_Block_详解.md
00_Z07_Y02_Kernel启动机制_详解.md
01_Z3_Y01_BF16_vs_FP16_详解.md
01_Z3_Y02_KV_Cache工作原理详解.md
01_Z3_Y03_精度不敏感权重_详解.md
```

#### 2.4 Day 系列文档

**格式**：`00_Day[数字]_[描述]_学习指南.md` 或 `DAY[数字]_[主题代码]_[描述].md`

**规则**：
- 用于按日期组织的学习文档
- 可以是 `Day03_` 或 `DAY04_` 格式

**示例**：
```
00_Day03_Benchmark_and_Profiling_学习指南.md
DAY04_Z1_SGLang_CI_CD系统详解.md
```

**特点**：
- ✅ 层级清晰，支持多级子文档
- ✅ 主题分类明确（Z 系列）
- ✅ 支持日期组织（Day 系列）

### 3. `md/AI_system_design/` 目录 - 字母分类系统

**用途**：AI 系统设计相关的文档

**命名格式**：
```
[字母][数字]_[描述].md
```

**规则**：
- 使用**字母+数字**组合表示分类
- 字母表示文档类型：
  - `A` = 核心内容/主要概念
  - `B` = 问答/面试题
  - `C` = 策略/方法
  - `D` = 计划/指南
- 数字从 `01` 开始递增

**示例**：
```
A01_LLM推理系统设计.md
A02_请求队列管理与批处理决策.md
B01_LLM推理系统设计面试问答.md
C01_面试心态与策略指南.md
D01_4-7天刷题冲刺计划.md
AI_System_Design面试准备指南.md  # 特殊：总览文档
```

**特点**：
- ✅ 分类明确（A/B/C/D）
- ✅ 适合面试准备和系统设计学习

---

### 4. `coding_practice/system_design/` 目录 - 多层级系统

**用途**：编码实践和系统设计项目

**命名格式**：
```
Day[数字]_[A-Z][数字]_[A-Z][数字]_[A-Z][数字]_[描述].md
Day[数字]_[A-Z][数字]_[A-Z][数字]_[A-Z][数字]_[D数字]_[描述].md
```

**规则**：
- 使用**多层级编号**：`Day02_A1_B4_C2_`
- 层级结构：`Day[数字]` → `[字母][数字]` → `[字母][数字]` → `[字母][数字]`
- 子文档使用 `D[数字]_` 后缀：`Day02_A1_B4_C2_D1_`, `Day02_A1_B4_C2_D2_`

**示例**：
```
Day02_A1_B4_C2_LLM应用框架对比_LangChain_AutoGen_MetaDeck.md
Day02_A1_B4_C2_D1_LangChain性能优化详解.md
Day02_A1_B4_C2_D2_传感器医疗项目中向量数据库使用详解.md
Day02_A1_B4_C1_云数据库vs自建数据库_Trade_off详解.md
```

**特点**：
- ✅ 支持深度层级结构
- ✅ 适合复杂项目组织
- ✅ 支持多级子文档（D 系列）

## 📝 通用命名规则

### 描述部分规则

1. **使用下划线分隔**：多个单词用下划线连接
   - ✅ `Cache_Aware_Router前缀匹配详解.md`
   - ✅ `Benchmark_and_Profiling_学习指南.md`
   - ❌ `Cache Aware Router前缀匹配详解.md`（空格）

2. **中文和英文混合**：
   - ✅ 可以使用中文：`为什么重要.md`, `详解.md`, `学习指南.md`
   - ✅ 可以使用英文：`Benchmark_and_Profiling.md`, `LLM推理系统设计.md`
   - ✅ 可以混合：`SGLang_学习指南.md`

3. **常见后缀**：
   - `_详解.md` - 详细解释文档
   - `_学习指南.md` - 学习指南文档
   - `_学习路径.md` - 学习路径文档
   - `_详细分析.md` - 详细分析文档
   - `_指南.md` - 指南文档

### 编号规则

1. **数字编号**：
   - 从 `01` 或 `1` 开始（根据上下文）
   - 按创建顺序递增
   - 预留编号空间（如 `13_01_`, `13_02_`, `13_03_`）

2. **字母编号**：
   - 使用大写字母：`A`, `B`, `C`, `D`, `Z`, `Y`
   - 从 `01` 或 `1` 开始（根据上下文）

3. **多层级编号**：
   - 使用下划线分隔：`Day02_A1_B4_C2_`
   - 保持层级一致性

## 📋 文件创建检查清单

创建新文件时，检查以下事项：

### 1. 选择正确的目录
- [ ] `md/` - 核心学习文档
- [ ] `llm study sglang_yc01252026/` - 系统化学习
- [ ] `md/AI_system_design/` - 系统设计
- [ ] `coding_practice/` - 编码实践

### 2. 选择正确的命名格式
- [ ] 数字编号系统（`md/`）
- [ ] Z/Y 系列系统（`llm study sglang_yc01252026/`）
- [ ] 字母分类系统（`md/AI_system_design/`）
- [ ] 多层级系统（`coding_practice/`）

### 3. 检查命名规范
- [ ] 前缀正确（`00_`, `01_`, `A01_`, `Day02_` 等）
- [ ] 编号连续（不跳过编号，除非有意预留）
- [ ] 描述清晰（使用下划线，避免空格）
- [ ] 后缀合适（`_详解.md`, `_学习指南.md` 等）

### 4. 检查层级关系
- [ ] 如果是子文档，使用正确的父文档编号
- [ ] 如果是多层级，保持层级一致性
- [ ] 如果是日期文档，使用 `Day` 或 `DAY` 前缀

## 🔄 文件更新规则

### 重命名文件

1. **保持编号不变**：除非是重大结构调整
2. **更新描述**：如果内容发生重大变化，可以更新描述部分
3. **更新索引**：如果重命名，更新相关索引文档

### 创建子文档

1. **使用父文档编号**：子文档使用父文档的编号作为前缀
2. **添加子编号**：使用 `_01_`, `_02_` 等子编号
3. **更新父文档**：在父文档中添加子文档的链接

### 删除文件

1. **保留编号**：删除文件后，编号可以保留（避免重新编号）
2. **更新索引**：从相关索引文档中移除链接
