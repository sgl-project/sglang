## 🎯 AI Infra / LLM Infra 工程师的主流工作

基于 SGLang 的开发工作和行业趋势，AI Infra / LLM Infra 工程师的主要工作包括：

### 1. **推理引擎开发与优化** 🚀

#### a) 性能优化

  - 优化模型 Forward Pass
  - 实现高效的批处理（Batching）
  - 优化内存管理（KV Cache、Paged Attention）

  - 推测解码（Speculative Decoding）
- **技能要求**:
  - CUDA/ROCm 编程
  - 深度学习框架（PyTorch, JAX）
  - 性能分析工具（Nsight, PyTorch Profiler）
  - 算法优化能力

#### b) 调度器开发
- **目标**: 高效调度请求，最大化资源利用率
- **工作内容**:
  - 连续批处理（Continuous Batching）

### 🎯 如何从 Bug 修复者成长为全面的 AI Infra 工程师？

#### **阶段 1: 深度理解（当前阶段）**

**在修复 Bug 时，要问自己**:
1. **这个 Bug 的根本原因是什么？**
   - 不仅仅是修复症状
   - 理解为什么会出现这个问题
   - 是否暴露了系统设计的问题？

2. **这个 Bug 影响的系统范围是什么？**
   - 是局部问题还是全局问题？
   - 修复后是否会影响其他功能？
   - 是否有更好的解决方案？

3. **系统的整体架构是怎样的？**
   - 修复 Bug 时，理解它在整个系统中的位置
   - 阅读相关代码，理解设计思路
   - 学习为什么系统是这样设计的

**具体行动**:
```python
# ❌ 不好的修复方式：只修复表面问题
def fix_bug():
    return None  # 简单返回，不深入理解

# ✅ 好的修复方式：深入理解系统
def fix_bug():
    # 1. 理解问题的根本原因
    root_cause = analyze_root_cause()
    
    # 2. 理解系统的设计意图
    design_intent = understand_system_design()
    
    # 3. 找到最合适的解决方案
    solution = find_best_solution(root_cause, design_intent)
    
    # 4. 确保不影响其他功能
    verify_no_regression(solution)
    
    return solution
```
