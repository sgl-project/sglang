## SQL 四大支柱（80% 题目都是拼凑）

### 核心策略

**不要去记具体的业务场景**（什么员工工资、体育馆人流量），**只记逻辑模型**。

掌握了这 4 个组件，80% 的 SQL 题都是它们的拼凑。

### 组件 A：排名与 Top N（窗口函数）

#### 核心函数

```
ROW_NUMBER(): 连续排名（1, 2, 3, 4...）
RANK(): 并列排名（1, 2, 2, 4...）
DENSE_RANK(): 密集排名（1, 2, 2, 3...）
```

#### 典型场景

```
- 谁是第一名？
- 每个部门的前三名？
- 每个类别的前 N 名？
```

#### 模板代码

```sql
-- 模板：每个部门的前三名
SELECT *
FROM (
    SELECT 
        *,
        ROW_NUMBER() OVER (PARTITION BY department ORDER BY salary DESC) as rn
    FROM employees
) t
WHERE rn <= 3;
```

#### 经典题目

```
1. LeetCode 185: Department Top Three Salaries
2. LeetCode 178: Rank Scores
3. LeetCode 177: Nth Highest Salary
```

---

### 组件 B：连续性问题（自连接或偏移）

#### 核心函数

```
LEAD(col, n): 向前偏移 n 行
LAG(col, n): 向后偏移 n 行
ROW_NUMBER() - 序列值: 判断连续性
```

#### 典型场景

```
- 连续 3 天活跃的用户？
- 订单增长的趋势？
- 连续登录的用户？
```

#### 模板代码

```sql
-- 模板：连续 3 天活跃的用户
SELECT DISTINCT user_id
FROM (
    SELECT 
        user_id,
        date,
        ROW_NUMBER() OVER (PARTITION BY user_id ORDER BY date) as rn,
        date - INTERVAL rn DAY as group_id
    FROM user_activity
) t
GROUP BY user_id, group_id
HAVING COUNT(*) >= 3;
```

#### 经典题目

```
1. LeetCode 180: Consecutive Numbers
2. LeetCode 601: Human Traffic of Stadium
3. LeetCode 1454: Active Users
```

### 组件 C：时间维度的转换

#### 核心函数

```
DATE_FORMAT(date, '%Y-%m'): 格式化日期
DATEDIFF(date1, date2): 日期差
TIMESTAMP_DIFF(timestamp1, timestamp2, unit): 时间戳差
DATE_ADD(date, INTERVAL n DAY): 日期加减
```

#### 典型场景

```
- 次日留存率？
- 月环比增长？
- 每周活跃用户？
```

#### 模板代码

```sql
-- 模板：次日留存率
SELECT 
    DATE_FORMAT(login_date, '%Y-%m-%d') as date,
    COUNT(DISTINCT user_id) as total_users,
    COUNT(DISTINCT CASE 
        WHEN next_login_date = DATE_ADD(login_date, INTERVAL 1 DAY) 
        THEN user_id 
    END) as retained_users,
    COUNT(DISTINCT CASE 
        WHEN next_login_date = DATE_ADD(login_date, INTERVAL 1 DAY) 
        THEN user_id 
    END) * 100.0 / COUNT(DISTINCT user_id) as retention_rate
FROM (
    SELECT 
        user_id,
        login_date,
        LEAD(login_date) OVER (PARTITION BY user_id ORDER BY login_date) as next_login_date
    FROM user_logins
) t
GROUP BY DATE_FORMAT(login_date, '%Y-%m-%d');
```

#### 经典题目

```
1. LeetCode 1098: Unpopular Books
2. LeetCode 1126: Active Businesses
3. 各种留存率、增长率题目
```

### 组件 D：条件聚合（CASE WHEN + SUM）

#### 核心模式

```
CASE WHEN condition THEN value ELSE 0 END
SUM(CASE WHEN condition THEN value ELSE 0 END)
COUNT(CASE WHEN condition THEN 1 END)
```

#### 典型场景

```
- 把行转成列？
- 计算特定条件的比例（比如已支付订单占比）？
- 多条件统计？
```

#### 模板代码

```sql
-- 模板：行转列（订单状态统计）
SELECT 
    DATE_FORMAT(order_date, '%Y-%m') as month,
    SUM(CASE WHEN status = 'paid' THEN 1 ELSE 0 END) as paid_orders,
    SUM(CASE WHEN status = 'pending' THEN 1 ELSE 0 END) as pending_orders,
    SUM(CASE WHEN status = 'cancelled' THEN 1 ELSE 0 END) as cancelled_orders,
    SUM(CASE WHEN status = 'paid' THEN amount ELSE 0 END) as paid_amount
FROM orders
GROUP BY DATE_FORMAT(order_date, '%Y-%m');
```

#### 经典题目

```
1. LeetCode 1179: Reformat Department Table
2. LeetCode 1193: Monthly Transactions I
3. LeetCode 1393: Capital Gain/Loss
```

## Python 算法：性价比最高的 3 类

### 核心策略

**数据工程师面试，算法通常不会考那种弯弯绕绕的技巧。**

你只需要掌握这 3 类，其他的（动态规划、复杂的图、二叉树旋转）**战略性放弃**。

### 类别 1：双指针/滑动窗口

#### 核心模式

```
双指针:
  - 左右指针（数组两端）
  - 快慢指针（链表）
  - 同向指针（滑动窗口）

滑动窗口:
  - 固定窗口大小
  - 可变窗口大小
  - 维护窗口内的状态
```

#### 典型场景

```
- 处理数组、字符串
- 子串、子数组问题
- 两数之和、三数之和
```

#### 模板代码

```python
# 模板：滑动窗口（固定大小）
def sliding_window_fixed(nums, k):
    window_sum = sum(nums[:k])
    max_sum = window_sum
    
    for i in range(k, len(nums)):
        window_sum = window_sum - nums[i-k] + nums[i]
        max_sum = max(max_sum, window_sum)
    
    return max_sum

# 模板：滑动窗口（可变大小）
def sliding_window_variable(s, target):
    left = 0
    window = {}
    result = []
    
    for right in range(len(s)):
        # 扩展窗口
        window[s[right]] = window.get(s[right], 0) + 1
        
        # 收缩窗口（满足条件时）
        while is_valid(window, target):
            # 更新结果
            result.append(s[left:right+1])
            
            # 收缩
            window[s[left]] -= 1
            if window[s[left]] == 0:
                del window[s[left]]
            left += 1
    
    return result
```

#### 经典题目

```
1. LeetCode 3: Longest Substring Without Repeating Characters
2. LeetCode 76: Minimum Window Substring
3. LeetCode 209: Minimum Size Subarray Sum
4. LeetCode 239: Sliding Window Maximum
5. LeetCode 1: Two Sum
6. LeetCode 15: 3Sum
```

### 类别 2：哈希表（Dict/Set）

#### 核心模式

```
解决所有"查找、去重、统计次数"的问题

Dict:
  - 统计频率
  - 快速查找
  - 映射关系

Set:
  - 去重
  - 快速判断存在性
  - 集合运算
```

#### 典型场景

```
- 统计字符/数字出现次数
- 快速查找（O(1)）
- 去重
- 两数之和、三数之和
```

#### 模板代码

```python
# 模板：统计频率
def count_frequency(nums):
    freq = {}
    for num in nums:
        freq[num] = freq.get(num, 0) + 1
    return freq

# 模板：快速查找
def two_sum(nums, target):
    seen = {}
    for i, num in enumerate(nums):
        complement = target - num
        if complement in seen:
            return [seen[complement], i]
        seen[num] = i
    return []

# 模板：去重
def remove_duplicates(nums):
    seen = set()
    result = []
    for num in nums:
        if num not in seen:
            seen.add(num)
            result.append(num)
    return result
```

#### 经典题目

```
1. LeetCode 1: Two Sum
2. LeetCode 49: Group Anagrams
3. LeetCode 128: Longest Consecutive Sequence
4. LeetCode 217: Contains Duplicate
5. LeetCode 242: Valid Anagram
```

### 类别 3：基础排序/查找逻辑

#### 核心模式

```
排序:
  - 内置排序（sorted()）
  - 自定义排序（key 函数）
  - 部分排序

查找:
  - 二分查找
  - 线性查找
  - 查找边界
```

#### 典型场景

```
- 排序数组
- 查找元素
- 查找边界
- Top K 问题
```

#### 模板代码

```python
# 模板：二分查找
def binary_search(nums, target):
    left, right = 0, len(nums) - 1
    
    while left <= right:
        mid = (left + right) // 2
        if nums[mid] == target:
            return mid
        elif nums[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    
    return -1

# 模板：自定义排序
def custom_sort(items):
    # 按多个条件排序
    return sorted(items, key=lambda x: (x[0], -x[1]))

# 模板：Top K
def top_k(nums, k):
    return sorted(nums, reverse=True)[:k]
    # 或使用堆：heapq.nlargest(k, nums)
```

#### 经典题目

```
1. LeetCode 704: Binary Search
2. LeetCode 35: Search Insert Position
3. LeetCode 34: Find First and Last Position
4. LeetCode 215: Kth Largest Element
5. LeetCode 56: Merge Intervals
```

### 战略性放弃的内容

```
❌ 动态规划（DP）
  - 除非明确要求，否则放弃
  - DE 岗位考 DP 的概率极低

❌ 复杂的图算法
  - DFS/BFS 基础可以了解
  - 但复杂图算法放弃

❌ 二叉树旋转
  - 除非明确要求，否则放弃
  - DE 岗位考树算法的概率极低

❌ 复杂的字符串算法
  - KMP、Manacher 等
  - 除非明确要求，否则放弃
```

**策略**：
- 如果遇到这些题目，**认栽**
- 但实际 DE 岗位考这些的概率**极低**
- 把时间花在 SQL 和系统设计上更值得

#### LeetCode 253: Meeting Rooms II ⭐⭐⭐

**为什么重要**：
- 对应资源分配问题（GPU 内存、KV Cache）
- 需要在有限资源下分配多个请求

**关键思路**：
```python
def minMeetingRooms(intervals):
    # 按开始时间排序（对应请求到达时间）
    intervals.sort(key=lambda x: x[0])
    
    # 使用最小堆存储结束时间（对应资源释放时间）
    heap = []
    
    for start, end in intervals:
        # 如果最早结束的会议已经结束，复用房间
        # 对应：如果 GPU 资源已释放，分配给新请求
        if heap and heap[0] <= start:
            heapq.heappop(heap)
        
        # 分配新房间（对应：分配 GPU 资源）
        heapq.heappush(heap, end)
    
    return len(heap)  # 需要的最大资源数
```

**面试回答话术**：
> "这题和 GPU 资源分配很像：我们需要在有限的 GPU 内存（会议室）下，分配多个请求（会议）。就像 SGLang 的调度器需要在有限的 KV Cache 内存下分配多个请求一样。"

#### LeetCode 239: Sliding Window Maximum ⭐⭐⭐

**为什么重要**：
- 对应动态批处理超时问题
- 需要在时间窗口内维护最大值

```python
def maxSlidingWindow(nums, k):
    # 使用双端队列维护窗口内的最大值
    # 对应：在时间窗口内维护优先级最高的请求
    dq = deque()
    result = []
    
    for i in range(len(nums)):
        # 移除窗口外的元素
        while dq and dq[0] < i - k + 1:
            dq.popleft()
        
        # 维护递减队列
        while dq and nums[dq[-1]] < nums[i]:
            dq.pop()
        
        dq.append(i)
        
        if i >= k - 1:
            result.append(nums[dq[0]])
    
    return result
```

#### LeetCode 1834: Single-Threaded CPU ⭐⭐⭐

**为什么重要**：
- 直接对应调度器实现
- 需要处理优先级和等待时间

**关键思路**：
```python
def getOrder(tasks):
    # 按到达时间排序
    tasks = sorted([(t[0], t[1], i) for i, t in enumerate(tasks)])
    
    # 使用最小堆存储可执行的任务（按处理时间、索引）
    heap = []
    time = 0
    result = []
    i = 0
    
    while i < len(tasks) or heap:
        # 添加所有到达的任务
        while i < len(tasks) and tasks[i][0] <= time:
            heapq.heappush(heap, (tasks[i][1], tasks[i][2]))
            i += 1
        
        if heap:
            # 执行最短的任务（对应：优先级最高的请求）
            duration, idx = heapq.heappop(heap)
            time += duration
            result.append(idx)
        else:
            # 如果没有可执行的任务，跳到下一个任务到达时间
            time = tasks[i][0]
    
    return result
```

**面试回答话术**：

#### LeetCode 146: LRU Cache ⭐⭐⭐

**为什么重要**：

- 理解 LRU 的实现原理

**关键思路**：
```python
class LRUCache:
    def __init__(self, capacity):
        self.capacity = capacity
        self.cache = {}  # {key: node}
        self.head = Node(0, 0)
        self.tail = Node(0, 0)
        self.head.next = self.tail
        self.tail.prev = self.head
    
    def get(self, key):
        if key in self.cache:
            node = self.cache[key]
            self._move_to_head(node)
            return node.value
        return -1
    
    def put(self, key, value):
        if key in self.cache:
            node = self.cache[key]
            node.value = value
            self._move_to_head(node)
        else:
            if len(self.cache) >= self.capacity:
                # 移除最久未使用的（对应：KV Cache 替换）
                self._remove_tail()
            node = Node(key, value)
            self.cache[key] = node
            self._add_to_head(node)
```

**面试回答话术**：
> "这题和 KV Cache 的缓存替换策略很像：当内存不足时，我们需要移除最久未使用的请求。就像 SGLang 的 KV Cache 管理器在内存不足时使用 LRU 替换策略一样。"

## 刷题技巧

### 1. 每道题的练习流程

```
步骤 1: 理解题目（5 分钟）
  - 仔细读题
  - 理解输入输出
  - 识别关键约束

步骤 2: 说出思路（10 分钟）
  - 不要直接写代码
  - 先说出思路
  - 解释算法选择

步骤 3: 写代码（15-20 分钟）
  - 边写边说
  - 解释每一步
  - 主动测试边界情况

步骤 4: 测试和调试（10 分钟）
  - 用测试用例验证
  - 如果发现 bug，主动修复
  - 展示调试过程

步骤 5: 优化（5 分钟）
  - 讨论时间/空间复杂度
  - 讨论优化方案
  - 解释权衡
```

### 2. 准备面试话术

**每道题都要准备**：

```
1. 思路解释:
   "我的思路是...，因为..."

2. 算法选择:
   "我选择使用...，因为..."

3. 复杂度分析:
   "时间复杂度是 O(...)，因为..."
   "空间复杂度是 O(...)，因为..."

4. 与实际系统的联系:
   "这题和...很像，因为..."

5. 优化方案:
   "如果优化，我会...，权衡是..."
```

### 3. 如果不会做的处理

```
标准流程:
  1. "这是一个很好的问题。让我先理解一下..."
  2. "我的初步思路是...，但不确定是否正确"
  3. "能否给我一些提示？"
  4. "让我尝试一个简化版本..."
  5. "如果给我更多时间，我会这样思考..."
```

### 4. 如果发现 bug 的处理

```
标准流程:
  1. "让我检查一下这个逻辑..."
  2. "我发现这里可能有问题..."
  3. "让我用测试用例验证一下..."
  4. "好的，我修复了这个 bug..."
  5. "这个 bug 的原因是...，我学到了..."
```

## 核心策略总结

### 1. 质量 > 数量

```
❌ 不要:
  - 刷 100 道题但都不熟练
  - 死记硬背代码
  - 追求速度和完美

✅ 要:
  - 精刷 15-20 道经典题目
  - 每道题都要能解释思路
  - 练习边写边说
```

### 2. 模式 > 题目

```
理解算法模式:
  - 优先级队列
  - 贪心算法
  - 滑动窗口
  - 双指针
  - 数据结构设计

举一反三:
  - 一道题 → 一类题
  - 理解本质 → 灵活应用
```

### 3. 沟通 > 完美

```
面试官更看重:
  ✅ 思路清晰
  ✅ 能解释自己的想法
  ✅ 能发现和修复 bug
  ✅ 能接受反馈
  ✅ 沟通顺畅

而不是:
  ❌ 是否完美做出
  ❌ 是否 bug-free
  ❌ 是否最优解
```

### 4. 系统设计 > 算法题（对于 AI Infra）

```
AI Infra 岗位评估权重:
  - 系统设计: 50%
  - 算法题: 30%
  - 其他: 20%

你的优势:
  ✅ 系统设计准备充分（A01, A02, B01）
  ✅ 有实际项目经验（SGLang）
  ✅ 理解实际系统架构

策略:
  - 算法题：展示思路和沟通能力
  - 系统设计：发挥你的优势
  - 综合：展示你的实际经验
```
