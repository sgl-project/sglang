# SGLang 环境配置指南

本指南将帮助你逐步配置 SGLang 的开发环境。

## 📋 配置前准备

在开始之前，请确认：
- ✅ Windows 10/11 (64位)
- ✅ 至少 16GB RAM（推荐 32GB+）
- ✅ 至少 50GB 可用磁盘空间
- ✅ 管理员权限（用于安装软件）

## 🔍 第一步：环境检查

运行环境检查脚本，查看当前环境状态：

```powershell
cd I:\yc_research\sglang\yc_self_learn
.\setup_environment.ps1
```

这个脚本会检查：
- Docker 是否安装和运行
- Python 是否安装
- pip 是否可用
- Git 是否安装（可选）
- NVIDIA GPU 是否可用（可选）
- 必要的目录是否存在
- HuggingFace Token 是否设置

## 📦 第二步：安装必要软件

### 2.1 安装 Docker Desktop（必需）

1. **下载 Docker Desktop**
   - 访问：https://www.docker.com/products/docker-desktop/
   - 下载 Docker Desktop Installer for Windows

2. **安装 Docker Desktop**
   - 运行安装程序
   - 按照提示完成安装
   - 安装完成后重启电脑（如果需要）

3. **配置 Docker Desktop**
   - 启动 Docker Desktop
   - 进入 Settings → General
   - ✅ 勾选 "Use the WSL 2 based engine"（推荐）
   - 如果提示安装 WSL 2，按照提示安装

4. **配置资源**（推荐）
   - Settings → Resources → Advanced
   - 设置：
     - CPUs: 至少 4 核
     - Memory: 至少 8GB（推荐 16GB+）
     - Disk image size: 至少 64GB

5. **验证安装**
   ```powershell
   docker --version
   docker run hello-world
   ```

### 2.2 安装 Python（如果还没有）

1. **下载 Python**
   - 访问：https://www.python.org/downloads/
   - 下载 Python 3.8 或更高版本（推荐 3.10+）

2. **安装 Python**
   - 运行安装程序
   - ✅ **重要**：勾选 "Add Python to PATH"
   - 选择 "Install Now" 或自定义安装

3. **验证安装**
   ```powershell
   python --version
   pip --version
   ```

### 2.3 安装 Git（可选但推荐）

1. **下载 Git**
   - 访问：https://git-scm.com/download/win
   - 下载 Git for Windows

2. **安装 Git**
   - 运行安装程序
   - 使用默认设置即可

3. **验证安装**
   ```powershell
   git --version
   ```

### 2.4 配置 NVIDIA GPU 支持（如果有 NVIDIA GPU）

1. **安装 NVIDIA 驱动**
   - 确保已安装最新的 NVIDIA 驱动
   - 访问：https://www.nvidia.com/Download/index.aspx

2. **安装 NVIDIA Container Toolkit**
   - 访问：https://docs.nvidia.com/datacenter/cloud-native/container-toolkit/install-guide.html#docker
   - 按照 Windows 安装指南操作

3. **验证 GPU 支持**
   ```powershell
   nvidia-smi
   docker run --rm --gpus all nvidia/cuda:12.0.0-base-ubuntu22.04 nvidia-smi
   ```

## 📁 第三步：创建目录结构

运行以下命令创建必要的目录：

```powershell
# 创建基础目录
$baseDir = "I:\yc_research"
New-Item -ItemType Directory -Force -Path "$baseDir\docker_data\huggingface"
New-Item -ItemType Directory -Force -Path "$baseDir\docker_data\models"
New-Item -ItemType Directory -Force -Path "$baseDir\docker_data\data"
New-Item -ItemType Directory -Force -Path "$baseDir\docker_data\outputs"
```

或者直接运行环境检查脚本，它会自动创建缺失的目录。

## 🔑 第四步：配置 HuggingFace Token

如果你需要使用 HuggingFace 上的模型（大多数情况需要）：

1. **获取 Token**
   - 访问：https://huggingface.co/settings/tokens
   - 登录你的 HuggingFace 账号
   - 点击 "New token"
   - 选择 "Read" 权限（或根据需要选择）
   - 复制生成的 token

2. **设置环境变量**

   **方法 1：临时设置（当前会话）**
   ```powershell
   $env:HF_TOKEN = "your-token-here"
   ```

   **方法 2：永久设置（推荐）**
   ```powershell
   # 在 PowerShell 中设置用户环境变量
   [System.Environment]::SetEnvironmentVariable("HF_TOKEN", "your-token-here", "User")
   
   # 重新打开 PowerShell 使设置生效
   ```

   **方法 3：在脚本中设置**
   - 编辑 `docker_scripts\run_sglang.ps1`
   - 修改 `$HF_TOKEN` 变量

3. **验证设置**
   ```powershell
   echo $env:HF_TOKEN
   ```

## 🐳 第五步：拉取 Docker 镜像

拉取 SGLang 的 Docker 镜像：

```powershell
# 拉取最新稳定版
docker pull lmsysorg/sglang:latest

# 或拉取开发版（用于开发）
docker pull lmsysorg/sglang:dev
```

**注意**：镜像较大（约 10-20GB），下载可能需要一些时间。

## 🆘 常见问题

### Q1: Docker 启动失败
**A:** 
- 确保 Docker Desktop 正在运行
- 检查是否有足够的磁盘空间
- 尝试重启 Docker Desktop

### Q2: GPU 不可用
**A:**
- 检查 NVIDIA 驱动是否最新
- 确认已安装 NVIDIA Container Toolkit
- 在 Docker Desktop 设置中启用 GPU 支持

### Q3: 端口被占用
**A:**
```powershell
# 检查端口占用
netstat -ano | findstr :30000

# 使用其他端口
.\run_sglang.ps1 -Port 30001
```

### Q4: 模型下载失败
**A:**
- 检查 HuggingFace Token 是否正确设置
- 检查网络连接
- 尝试手动下载模型到本地

### Q5: 内存不足
**A:**
- 在 Docker Desktop 中增加内存分配
- 使用更小的模型进行测试
- 关闭其他占用内存的程序
