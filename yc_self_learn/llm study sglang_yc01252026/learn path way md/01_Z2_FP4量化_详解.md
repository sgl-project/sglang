## 1. FP4量化概述

### 1.1 什么是FP4量化？

**FP4量化**是一种将浮点数从高精度（如FP16/BF16）压缩到4位浮点数的技术。

#### 1.1.1 基本概念

- **目的**: 减少模型内存占用，提升推理速度
- **压缩比**: 相比FP16，内存占用减少约**75%**（4位 vs 16位）
- **精度**: 相比INT4量化，FP4保持浮点数格式，精度损失更小
- **硬件支持**: Blackwell GPU支持FP4 Tensor Core，可以加速FP4矩阵运算

#### 1.1.2 FP4量化的优势

| 特性 | FP16 | INT4 | FP4 |
|------|------|------|-----|
| **位宽** | 16位 | 4位 | 4位 |
| **格式** | 浮点数 | 整数 | 浮点数 |
| **内存占用** | 100% | 25% | 25% |
| **精度** | 高 | 较低 | 中等（比INT4好） |
| **硬件支持** | 广泛 | 有限 | Blackwell GPU |

### 1.2 FP4量化的应用场景

- **模型权重量化**: 将LLM的权重从FP16量化到FP4
- **MoE模型**: 特别适合MoE架构，可以大幅减少权重存储
- **推理加速**: 利用Blackwell GPU的FP4 Tensor Core加速矩阵运算

## 2. FP4格式（E2M1）详解

### 2.1 E2M1格式定义

**E2M1**是FP4的一种格式，表示：
- **E2**: 2位指数（Exponent）
- **M1**: 1位尾数（Mantissa）
- **S**: 1位符号位（Sign）

**总位数**: 1位符号 + 2位指数 + 1位尾数 = **4位**

### 2.2 E2M1的位布局

```
FP4 (E2M1) 位布局:
┌─────┬─────┬─────┬─────┐
│  S  │ E1  │ E0  │  M  │
└─────┴─────┴─────┴─────┘
  1位   2位   1位
```

- **S (Sign)**: 符号位，0=正数，1=负数
- **E1, E0 (Exponent)**: 2位指数，范围0-3
- **M (Mantissa)**: 1位尾数，0或1

**参考文档**: [00_Z2_Y01_FP4格式设计详解.md](./00_Z2_Y01_FP4格式设计详解.md) ⭐ **详细讲解格式设计、硬件vs软件实现**

### 2.3 E2M1的数值表示

**E2M1格式可以表示的值**:

| 二进制 | 符号 | 指数 | 尾数 | 数值 |
|--------|------|------|------|------|
| 0000 | + | 0 | 0 | 0.0 |
| 0001 | + | 0 | 1 | 0.5 |
| 0010 | + | 1 | 0 | 1.0 |
| 0011 | + | 1 | 1 | 1.5 |
| 0100 | + | 2 | 0 | 2.0 |
| 0101 | + | 2 | 1 | 3.0 |
| 0110 | + | 3 | 0 | 4.0 |
| 0111 | + | 3 | 1 | 6.0 |
| 1000 | - | 0 | 0 | -0.0 |
| 1001 | - | 0 | 1 | -0.5 |
| ... | ... | ... | ... | ... |

**E2M1的最大值**: 6.0（二进制：0111）

### 2.4 E2M1的数值范围

```
E2M1数值范围:
-6.0  ──────────────── 0 ──────────────── 6.0
  ↑                      ↑                      ↑
最小值                 零值                  最大值
```

**关键值**:
- **最大值**: 6.0（`E2M1_max = 6.0`）
- **最小值**: -6.0
- **可表示的值**: 0, ±0.5, ±1.0, ±1.5, ±2.0, ±3.0, ±4.0, ±6.0

## 3. Scale计算

### 3.1 Scale的作用

**Scale（缩放因子）**用于将FP16的值映射到FP4的范围（-6.0到6.0）。

#### 3.1.1 Scale计算原理

```
FP16值范围: [-65504, 65504]  (FP16的最大值)
FP4值范围:  [-6.0, 6.0]      (E2M1的最大值)

Scale = FP16_max / FP4_max = FP16_max / 6.0
量化值 = FP16值 / Scale
```

### 3.2 Block-wise Scale

**关键概念**: 不是对整个tensor使用一个scale，而是对每个**block**（通常是16或32个元素）计算一个独立的scale。

#### 3.2.1 Block-wise Scale的优势

- **更精确**: 每个block使用自己的scale，可以更好地适应数据分布
- **减少精度损失**: 相比全局scale，block-wise scale可以保留更多信息

#### 3.2.2 Scale计算步骤

```python
# 伪代码示例
def compute_scale(input_block):
    # 1. 找到block中的最大值
    block_max = abs(input_block).max()
    
    # 2. 计算descale（反缩放因子）
    descale = block_max / E2M1_max  # E2M1_max = 6.0
    
    # 3. 计算scale（使用对数，存储为FP8）
    scale = log2(descale)
    scale = ceil(max(scale, -127))  # 限制范围
    
    # 4. 量化scale为FP8格式（用于存储）
    scale_fp8 = quantize_to_fp8(scale)
    
    return scale_fp8
```

## 4. Quantization（量化）过程

### 4.1 量化流程

```
FP16值 ──> 除以Scale ──> 映射到E2M1范围 ──> 转换为FP4 ──> 打包为uint8
```

### 4.2 量化步骤详解

#### 步骤1: 归一化到E2M1范围

```python
# 将FP16值除以scale，映射到[-6.0, 6.0]范围
normalized_value = fp16_value / scale
```

#### 步骤2: 映射到E2M1离散值

```python
# E2M1可以表示的值: [0, 0.5, 1.0, 1.5, 2.0, 3.0, 4.0, 6.0]
# 找到最接近的值
E2M1_bounds = [0.25, 0.75, 1.25, 1.75, 2.5, 3.5, 5]

def cast_to_fp4(value):
    sign = sign(value)
    abs_value = abs(value)
    
    # 找到abs_value在bounds中的位置
    ord_ = sum((abs_value - bounds) > 0)
    
    # 组合符号位和序号
    fp4_val = (sign_bit * 0b1000) + ord_
    return fp4_val
```

#### 步骤3: 打包为uint8

**关键**: 两个FP4值（各4位）打包成一个uint8（8位）

```python
def fuse_uint4_to_uint8(fp4_values):
    # fp4_values: [v0, v1, v2, v3, ...]
    # 打包为: [v1|v0, v3|v2, ...]
    
    left_side = fp4_values[0::2]   # 偶数索引: v0, v2, ...
    right_side = fp4_values[1::2]  # 奇数索引: v1, v3, ...
    
    # 右移4位，将奇数索引的值放在高4位
    packed = (right_side << 4) + left_side
    
    return packed  # uint8数组，长度是原来的一半
```

### 4.3 SGLang中的量化实现

**代码位置**: `python/sglang/srt/layers/quantization/mxfp4_tensor.py`

```python
@classmethod
def quantize(cls, input: torch.Tensor, block_size: Optional[int]) -> tuple:
    # 1. 计算scale
    input_amax = input.abs().max(dim=-1, keepdim=True).values
    descale = input_amax / cls.E2M1_max  # E2M1_max = 6.0
    e8m0_scale = torch.ceil(torch.maximum(torch.log2(descale), min_value))
    
    # 2. 归一化
    input = input / torch.exp2(e8m0_scale)
    
    # 3. 转换为FP4
    input_q = cast_fp4(input)
    
    # 4. 打包为uint8
    input_q = fuse_uint4_to_uint8(input_q)
    
    # 5. 量化scale为uint8
    e8m0_scale = (e8m0_scale + 127).to(torch.uint8)
    
    return quantized_data, e8m0_scale
```

## 5. Dequantization（反量化）过程

### 5.1 反量化流程

```
FP4 (uint8) ──> 解包为两个FP4值 ──> 转换为FP16 ──> 乘以Scale ──> FP16值
```

### 5.2 反量化步骤详解

#### 步骤1: 解包uint8

```python
def unfuse_uint8_to_uint4(packed_uint8):
    # packed_uint8: [v1|v0, v3|v2, ...]
    # 解包为: [v0, v1, v2, v3, ...]
    
    left_side = packed_uint8 & 0x0F   # 低4位（偶数索引）
    right_side = (packed_uint8 >> 4) & 0x0F  # 高4位（奇数索引）
    
    # 交错组合
    result = torch.zeros(shape * 2)
    result[0::2] = left_side   # 偶数索引
    result[1::2] = right_side  # 奇数索引
    
    return result
```

#### 步骤2: 转换为FP16

```python
# E2M1值映射表
E2M1_values = [0, 0.5, 1, 1.5, 2, 3, 4, 6]

def cast_from_fp4(fp4_val):
    sign_bit = (fp4_val >> 3) & 0x1
    ord_ = fp4_val & 0x7
    
    value = E2M1_values[ord_]
    if sign_bit == 1:
        value = -value
    
    return value
```

#### 步骤3: 乘以Scale恢复原始范围

```python
# 恢复原始FP16值
fp16_value = fp4_value * scale
```

### 5.3 SGLang中的反量化实现

**代码位置**: `python/sglang/srt/layers/quantization/mxfp4_tensor.py`

```python
@classmethod
def dequantize(cls, quantized_data, dtype: torch.dtype, scale, block_sizes):
    # 1. 解包uint8
    fp4_values = unfuse_uint8_to_uint4(quantized_data)
    
    # 2. 转换为FP16值
    fp16_values = cast_from_fp4(fp4_values)
    
    # 3. 乘以scale恢复原始范围
    # scale需要从FP8格式恢复
    scale_fp32 = dequantize_scale_from_fp8(scale)
    result = fp16_values * scale_fp32
    
    return result
```

## ✅ 总结

### 核心要点

1. **FP4格式**: E2M1格式（2位指数+1位尾数+1位符号），可以表示8个离散值
2. **Scale计算**: Block-wise scale，每个block（16-32元素）独立计算scale
3. **量化过程**: FP16 → 归一化 → E2M1映射 → 打包为uint8
4. **反量化过程**: uint8解包 → E2M1转换 → 乘以scale → FP16

### 关键理解

- ✅ **FP4 = 4位浮点数**: 相比INT4，保持浮点数格式，精度更好
- ✅ **Block-wise Scale**: 每个block独立scale，减少精度损失
- ✅ **打包存储**: 两个FP4值打包成一个uint8，节省存储
- ✅ **硬件加速**: Blackwell GPU支持FP4 Tensor Core，可以加速计算
