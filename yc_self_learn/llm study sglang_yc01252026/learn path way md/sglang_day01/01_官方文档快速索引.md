# 官方文档快速索引

## 🎯 按优先级分类的官方文档

---

## ⭐⭐⭐ 最高优先级 - 必读文档

### LLM和Transformer基础
1. **[Attention Is All You Need (论文)](https://arxiv.org/abs/1706.03762)**
   - Transformer架构的原始论文
   - 理解Self-Attention机制
   - 学习时间: 2-3小时

2. **[The Illustrated Transformer](https://jalammar.github.io/illustrated-transformer/)**
   - Transformer的可视化解释
   - 非常适合理解概念
   - 学习时间: 1小时

3. **[How GPT Works - Andrej Karpathy (视频)](https://www.youtube.com/watch?v=wjZofJX0v4M)**
   - LLM推理的直观解释
   - 视频教程，易于理解
   - 学习时间: 1小时

### GPU和CUDA
4. **[CUDA C++ Programming Guide](https://docs.nvidia.com/cuda/cuda-c-programming-guide/)**
   - NVIDIA官方CUDA文档
   - GPU编程的权威指南
   - 学习时间: 10-15小时（完整阅读）

5. **[An Even Easier Introduction to CUDA](https://developer.nvidia.com/blog/even-easier-introduction-cuda/)**
   - CUDA入门教程
   - 适合初学者
   - 学习时间: 2-3小时

6. **[Programming Tensor Cores in CUDA 9](https://developer.nvidia.com/blog/programming-tensor-cores-cuda-9/)**
   - Tensor Core编程指南
   - Issue #17526相关
   - 学习时间: 2-3小时

### 量化技术
7. **[FP8 Formats for Deep Learning (论文)](https://arxiv.org/abs/2209.05433)**
   - FP8量化的原始论文
   - Issue #17526核心
   - 学习时间: 2-3小时

8. **[PyTorch Quantization](https://pytorch.org/docs/stable/quantization.html)**
   - PyTorch量化官方文档
   - 理解量化实现
   - 学习时间: 3-4小时

### 性能优化
9. **[Flash Attention (论文)](https://arxiv.org/abs/2205.14135)**
   - Flash Attention原始论文
   - 理解高效的Attention实现
   - 学习时间: 2-3小时

10. **[Flash Attention 2 (论文)](https://arxiv.org/abs/2307.08691)**
    - Flash Attention的改进版
    - 更高效的实现
    - 学习时间: 1-2小时

### SGLang
11. **[SGLang官方文档](https://docs.sglang.ai/)**
    - SGLang的完整文档
    - 框架使用指南
    - 学习时间: 持续参考

12. **[SGLang GitHub](https://github.com/sgl-project/sglang)**
    - 源代码
    - Issue跟踪
    - 学习时间: 持续参考

**📌 SGLang 生态系统关系说明：**
- **SGLang**：由 LMSYS 团队开发的高性能推理/服务框架
- **Megatron**：由 NVIDIA 开发的分布式训练框架（用于训练阶段）
- **slime**：由 THUDM（清华大学）开发的后训练框架，整合了 Megatron（训练）和 SGLang（推理）
  - 架构：`slime = Megatron（训练引擎）+ SGLang（推理引擎）的协调层`
  - 用途：用于 GLM-4.6 等模型的后训练流程
  - GitHub: https://github.com/THUDM/slime

## 📖 文档阅读建议

### 阅读顺序
1. **先读可视化/入门文档** - 建立直观理解
2. **再读论文** - 深入理解原理
3. **最后看代码** - 理解实现细节

### 阅读技巧
1. **快速浏览** - 先看摘要和结论
2. **重点深入** - 对关键部分仔细阅读
3. **做笔记** - 记录关键概念和公式
4. **实践验证** - 边读边实践

### 时间分配
- **论文**: 2-3小时/篇
- **官方文档**: 持续参考
- **代码**: 结合Issue理解
