# 双线作战策略：LLM 与 Data 面试同时准备

## 🎯 你的困境

```
现实情况：
  ✅ 面试岗位五花八门（LLM + Data）
  ❌ 时间有限（每天 2-3 小时）
  ❌ 不可能"什么都会"
  
核心问题：
  如何用最少的时间，同时覆盖两种面试？
```

---

## 💡 核心策略：找到共同点，差异化准备

### 两种面试的共同点（80% 重叠）

```
✅ 都需要：
  - LeetCode 基础题（哈希表、双指针、排序）
  - 系统设计思维（架构能力）
  - 业务逻辑转换能力
  - 沟通表达能力

✅ 都看重：
  - 解决问题的思路
  - 学习能力
  - 工程思维
```

### 两种面试的差异点（20% 不同）

```
LLM 面试（AI Infra）：
  ⭐⭐⭐⭐⭐ SGLang 系统理解（调度、KV Cache）
  ⭐⭐⭐⭐   吞吐量 vs 延迟权衡
  ⭐⭐⭐     Transformer/Attention 基础
  ⭐⭐      LeetCode Medium（够用）
  ⭐        SQL/Pandas（几乎不需要）

Data 面试（Data Engineer）：
  ⭐⭐⭐⭐⭐ SQL 四大支柱（排名、连续、时间、聚合）
  ⭐⭐⭐⭐   Pandas 常用操作（groupby、merge、pivot）
  ⭐⭐⭐    业务逻辑转换（需求 → SQL）
  ⭐⭐      LeetCode Easy/Medium（够用）
  ⭐       LLM 了解（加分项，不是必须）
```

---

## 🚀 双线作战的"最小化准备"策略

### 策略 1：共同基础（每天必做）

```
时间：每天 30-40 分钟
内容：
  ✅ LeetCode 基础题（哈希表、双指针、排序）
     - 你已经有了哈希表题目！
     - 再补充双指针、排序各 5-8 道
     - 总共 20-30 道经典题，覆盖常见模式
  
  ✅ 系统设计思维（每周 2-3 次）
     - LLM 推理系统设计
     - 数据管道设计
     - 两者有共同点：资源管理、吞吐量优化
```

### 策略 2：差异化准备（按面试时间分配）

```
面试前 2-3 天：
  - 如果是 LLM 面试 → 重点复习 SGLang 系统
  - 如果是 Data 面试 → 重点复习 SQL/Pandas
  
面试当天早上：
  - 快速过一遍对应方向的"够用"技能
  - 准备常见问题的回答
```

### 策略 3：建立"知识库"（一次性投入）

```
一次性准备（2-3 天）：
  ✅ LLM 方向知识库：
     - SGLang 核心模块总结（你已经有了！）
     - 调度系统详解
     - KV Cache 管理
     - RadixAttention 原理
  
  ✅ Data 方向知识库：
     - SQL 四大支柱模板（你已经有了！）
     - Pandas 常用操作速查表
     - 业务逻辑转换案例
  
  ✅ 共同基础库：
     - LeetCode 经典题解（20-30 道）
     - 系统设计模板
```

## 📅 每日时间分配（2-3 小时）

### 方案 A：没有明确面试时（日常准备）

```
晚上 2-3 小时：
```

```
  - 如果是 LLM 面试：
    * SGLang 核心模块复写
    * 调度系统详解
    * KV Cache 管理
    * 准备"为什么 SGLang 快"的回答
  
  - 如果是 Data 面试：
    * SQL 四大支柱各 3-4 道题
    * Pandas 常用操作练习
    * 业务逻辑转换案例

[30 分钟] 另一个方向应急准备
  - 快速过一遍"够用"技能
  - 准备"我不会，但我能快速学习"的回答

[30 分钟] 模拟面试
  - 自己提问自己回答
  - 练习表达思路
```

## 🎯 快速切换技巧

### 技巧 1：建立"面试前检查清单"

```

  - SGLang 调度系统（waiting_queue、get_new_batch）
  - KV Cache 管理（内存分配、释放）
  - RadixAttention 原理（前缀匹配、缓存）
  - 吞吐量 vs 延迟权衡

✅ 够用技能（10 分钟）：
  - LeetCode 基础题（哈希表、双指针）- 快速过一遍思路
  - SQL 基础（万一问到）- 知道窗口函数就行

✅ 常见问题准备（10 分钟）：
  - "为什么 SGLang 快？"
  - "如何优化吞吐量？"
  - "KV Cache 如何管理？"
```

#### Data 面试前检查清单（面试当天早上）

```
✅ 核心知识（30 分钟）：
  - SQL 四大支柱（排名、连续、时间、聚合）- 各 1-2 道题快速过
  - Pandas 常用操作（groupby、merge、pivot）- 快速过一遍
  - 业务逻辑转换案例

✅ 够用技能（10 分钟）：
  - LeetCode Easy/Medium（字符串、数组）- 快速过一遍思路

✅ 常见问题准备（10 分钟）：
```

### 技巧 3：建立"知识库速查表"

#### LLM 方向速查表（一页纸）

```
SGLang 核心：
  - 调度系统：waiting_queue → get_new_batch → process_batch
  - KV Cache：RadixCache、内存管理、前缀匹配
  - 优化：RadixAttention、Prefill-Decode 分离

常见问题：
  - 为什么快？→ RadixAttention + 调度优化
  - 如何优化吞吐量？→ Batching + KV Cache 复用
  - 如何优化延迟？→ 动态批处理 + 优先级队列
```

#### Data 方向速查表（一页纸）

```
SQL 四大支柱：
  - 排名：ROW_NUMBER()、RANK()、DENSE_RANK()
  - 连续：LAG()、LEAD()、自连接
  - 时间：DATE_TRUNC()、EXTRACT()、时间窗口
  - 聚合：SUM(CASE WHEN)、GROUP BY + HAVING

Pandas 常用：
  - groupby().agg()、merge()、pivot_table()
  - 数据清洗：drop_duplicates()、fillna()、dropna()
```

## 🎯 最终策略总结

### 核心原则

```
1. 共同基础（每天必做）
   - LeetCode 基础题（哈希表、双指针、排序）
   - 系统设计思维

2. 差异化准备（按面试时间分配）
   - 面试前 2-3 天：重点突破面试方向
   - 日常准备：主攻方向 + 另一个方向"够用"

3. 快速切换（建立检查清单和速查表）
   - LLM 面试前检查清单
   - Data 面试前检查清单
   - 知识库速查表

4. 展示学习能力（比"什么都会"更重要）
   - 准备"我不会，但我能快速学习"的回答
   - 展示解决问题的思路
```

### 时间分配

```
没有明确面试时（日常准备）：
  - 40%：共同基础（LeetCode）
  - 40%：主攻方向（LLM 或 Data，轮换）
  - 15%：另一个方向（保持"够用"）
  - 5%：  总结和反思

有明确面试时（面试前 2-3 天）：
  - 30%：共同基础（保持手感）
  - 50%：面试方向重点突破
  - 15%：另一个方向应急准备
  - 5%：  模拟面试
```
