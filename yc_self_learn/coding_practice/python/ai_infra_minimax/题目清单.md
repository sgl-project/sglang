# AI Infra LLM 面试刷题清单（MiniMax 专用）

## 📚 题目分类（按重要性和相关性排序）

---

## 🎯 第一优先级：调度与批处理相关（必刷）

### LeetCode 621: Task Scheduler (任务调度器)
**为什么有意义**：
- ✅ **调度器核心**：LLM 推理系统的 Scheduler 就是任务调度器
- ✅ **批处理思想**：如何合理安排任务，最大化吞吐量，类似 Continuous Batching
- ✅ **冷却时间**：模拟 GPU 执行时间和资源约束
- ✅ **面试高频**：直接对应推理系统的调度逻辑

**核心考点**：
- 贪心算法
- 优先级队列（堆）
- 时间窗口调度

---

### LeetCode 253: Meeting Rooms II (会议室 II)
**为什么有意义**：
- ✅ **资源分配**：类似 GPU 资源分配，如何用最少的资源处理最多请求
- ✅ **区间调度**：Prefill 和 Decode 的时序管理
- ✅ **负载均衡**：多 Worker 之间的任务分配

**核心考点**：
- 区间问题
- 扫描线算法
- 贪心策略

---

### LeetCode 295: Find Median from Data Stream (数据流的中位数)
**为什么有意义**：
- ✅ **动态统计**：Scheduler 需要实时统计延迟、吞吐量等指标
- ✅ **双堆结构**：维护有序集合，用于优先级调度
- ✅ **性能优化**：O(log n) 插入，O(1) 查询中位数

**核心考点**：
- 堆（大顶堆 + 小顶堆）
- 流式数据处理
- 动态维护有序集合

---

## 🔥 第二优先级：缓存与内存管理

### LeetCode 146: LRU Cache (LRU 缓存)
**为什么有意义**：
- ✅ **KV Cache 淘汰**：LRU 是 KV Cache 常用的淘汰策略
- ✅ **内存管理**：如何高效管理有限的 GPU 内存
- ✅ **O(1) 操作**：哈希表 + 双向链表，高效实现

**核心考点**：
- 哈希表 + 双向链表
- 设计数据结构
- O(1) 时间复杂度

---

### LeetCode 460: LFU Cache (LFU 缓存)
**为什么有意义**：
- ✅ **缓存策略**：LFU 是另一种缓存淘汰策略，vLLM 等系统使用
- ✅ **频率统计**：类似统计 KV Cache 访问频率
- ✅ **复合数据结构**：哈希表 + 堆/有序集合 + 链表

**核心考点**：
- 复杂数据结构设计
- 频率统计
- 多层哈希表

---

### LeetCode 380: Insert Delete GetRandom O(1) (O(1) 时间插入、删除和获取随机元素)
**为什么有意义**：
- ✅ **内存池管理**：KV Cache 内存池需要快速分配和回收
- ✅ **随机采样**：用于负载均衡、采样等场景
- ✅ **O(1) 操作**：哈希表 + 动态数组的组合

**核心考点**：
- 哈希表 + 数组
- O(1) 随机访问
- 删除优化

## 🎓 面试技巧

### 1. 题目理解
- 看到调度题 → 联想到 Continuous Batching
- 看到缓存题 → 联想到 KV Cache 管理
- 看到队列题 → 联想到请求队列

### 2. 算法选择
- **调度类**：优先考虑堆/优先级队列
- **缓存类**：哈希表 + 链表/数组
- **统计类**：哈希表 + 堆/桶排序

### 3. 优化方向
- **时间复杂度**：O(n²) → O(n)，用哈希表优化
- **空间复杂度**：用时间换空间，或者原地算法
- **实际应用**：结合 LLM 推理系统的实际场景
