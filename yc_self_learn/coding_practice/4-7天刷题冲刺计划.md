## SQL 四大支柱（80% 题目都是拼凑）

### 核心策略

**不要去记具体的业务场景**（什么员工工资、体育馆人流量），**只记逻辑模型**。

掌握了这 4 个组件，80% 的 SQL 题都是它们的拼凑。

### 组件 A：排名与 Top N（窗口函数）

#### 核心函数

```
ROW_NUMBER(): 连续排名（1, 2, 3, 4...）
RANK(): 并列排名（1, 2, 2, 4...）
DENSE_RANK(): 密集排名（1, 2, 2, 3...）
```

#### 典型场景

```
- 谁是第一名？
- 每个部门的前三名？
- 每个类别的前 N 名？
```

#### 模板代码

```sql
-- 模板：每个部门的前三名
SELECT *
FROM (
    SELECT 
        *,
        ROW_NUMBER() OVER (PARTITION BY department ORDER BY salary DESC) as rn
    FROM employees
) t
WHERE rn <= 3;
```

#### 经典题目

```
1. LeetCode 185: Department Top Three Salaries
2. LeetCode 178: Rank Scores
3. LeetCode 177: Nth Highest Salary
```

---

### 组件 B：连续性问题（自连接或偏移）

#### 核心函数

```
LEAD(col, n): 向前偏移 n 行
LAG(col, n): 向后偏移 n 行
ROW_NUMBER() - 序列值: 判断连续性
```

#### 典型场景

```
- 连续 3 天活跃的用户？
- 订单增长的趋势？
- 连续登录的用户？
```

#### 模板代码

```sql
-- 模板：连续 3 天活跃的用户
SELECT DISTINCT user_id
FROM (
    SELECT 
        user_id,
        date,
        ROW_NUMBER() OVER (PARTITION BY user_id ORDER BY date) as rn,
        date - INTERVAL rn DAY as group_id
    FROM user_activity
) t
GROUP BY user_id, group_id
HAVING COUNT(*) >= 3;
```

#### 经典题目

```
1. LeetCode 180: Consecutive Numbers
2. LeetCode 601: Human Traffic of Stadium
3. LeetCode 1454: Active Users
```

### 组件 C：时间维度的转换

#### 核心函数

```
DATE_FORMAT(date, '%Y-%m'): 格式化日期
DATEDIFF(date1, date2): 日期差
TIMESTAMP_DIFF(timestamp1, timestamp2, unit): 时间戳差
DATE_ADD(date, INTERVAL n DAY): 日期加减
```

#### 典型场景

```
- 次日留存率？
- 月环比增长？
- 每周活跃用户？
```

#### 模板代码

```sql
-- 模板：次日留存率
SELECT 
    DATE_FORMAT(login_date, '%Y-%m-%d') as date,
    COUNT(DISTINCT user_id) as total_users,
    COUNT(DISTINCT CASE 
        WHEN next_login_date = DATE_ADD(login_date, INTERVAL 1 DAY) 
        THEN user_id 
    END) as retained_users,
    COUNT(DISTINCT CASE 
        WHEN next_login_date = DATE_ADD(login_date, INTERVAL 1 DAY) 
        THEN user_id 
    END) * 100.0 / COUNT(DISTINCT user_id) as retention_rate
FROM (
    SELECT 
        user_id,
        login_date,
        LEAD(login_date) OVER (PARTITION BY user_id ORDER BY login_date) as next_login_date
    FROM user_logins
) t
GROUP BY DATE_FORMAT(login_date, '%Y-%m-%d');
```

#### 经典题目

```
1. LeetCode 1098: Unpopular Books
2. LeetCode 1126: Active Businesses
3. 各种留存率、增长率题目
```

#### 常见 SQL Medium 语法错误点（重点记忆）

```
❌ 窗口函数语法错误：
  - OVER() 后面忘记 PARTITION BY 或 ORDER BY
  - 忘记窗口函数不能直接在 WHERE 中使用，需要子查询
  - ROW_NUMBER() vs RANK() vs DENSE_RANK() 的语法混淆

❌ CTE 语法错误：
  - WITH ... AS () 的括号位置错误
  - 多个 CTE 之间忘记逗号
  - CTE 中引用其他 CTE 的顺序错误

❌ 子查询语法错误：
  - 忘记给子查询加别名
  - 子查询中 SELECT 字段数量不匹配
  - 相关子查询的 WHERE 条件写错

❌ CASE WHEN 语法错误：
  - CASE WHEN ... THEN ... ELSE ... END 的 END 忘记
  - CASE WHEN 在聚合函数中的位置错误
  - CASE WHEN 的条件判断逻辑错误

❌ 日期函数语法错误：
  - DATE_FORMAT() 的格式字符串错误
  - DATEDIFF() 的参数顺序错误
  - DATE_ADD() 的 INTERVAL 语法错误

❌ JOIN 语法错误：
  - LEFT JOIN vs INNER JOIN 的选择错误
  - ON 条件的字段名写错
  - 多个 JOIN 的顺序导致逻辑错误
```

## Python 算法：性价比最高的 3 类

### 核心策略

**数据工程师面试，算法通常不会考那种弯弯绕绕的技巧。**

你只需要掌握这 3 类，其他的（动态规划、复杂的图、二叉树旋转）**战略性放弃**。

### 类别 1：双指针/滑动窗口

#### 核心模式

```
双指针:
  - 左右指针（数组两端）
  - 快慢指针（链表）
  - 同向指针（滑动窗口）

滑动窗口:
  - 固定窗口大小
  - 可变窗口大小
  - 维护窗口内的状态
```

#### 典型场景

```
- 处理数组、字符串
- 子串、子数组问题
- 两数之和、三数之和
```

#### 模板代码

```python
# 模板：滑动窗口（固定大小）
def sliding_window_fixed(nums, k):
    window_sum = sum(nums[:k])
    max_sum = window_sum
    
    for i in range(k, len(nums)):
        window_sum = window_sum - nums[i-k] + nums[i]
        max_sum = max(max_sum, window_sum)
    
    return max_sum

# 模板：滑动窗口（可变大小）
def sliding_window_variable(s, target):
    left = 0
    window = {}
    result = []
    
    for right in range(len(s)):
        # 扩展窗口
        window[s[right]] = window.get(s[right], 0) + 1
        
        # 收缩窗口（满足条件时）
        while is_valid(window, target):
            # 更新结果
            result.append(s[left:right+1])
            
            # 收缩
            window[s[left]] -= 1
            if window[s[left]] == 0:
                del window[s[left]]
            left += 1
    
    return result
```

#### 经典题目

```
1. LeetCode 3: Longest Substring Without Repeating Characters
2. LeetCode 76: Minimum Window Substring
3. LeetCode 209: Minimum Size Subarray Sum
4. LeetCode 239: Sliding Window Maximum
5. LeetCode 1: Two Sum
6. LeetCode 15: 3Sum
```

### 类别 2：哈希表（Dict/Set）

#### 核心模式

```
解决所有"查找、去重、统计次数"的问题

Dict:
  - 统计频率
  - 快速查找
```

```python
# 模板：统计频率
def count_frequency(nums):
    freq = {}
    for num in nums:
        freq[num] = freq.get(num, 0) + 1
    return freq

# 模板：快速查找
def two_sum(nums, target):
    seen = {}
    for i, num in enumerate(nums):
        complement = target - num
        if complement in seen:
            return [seen[complement], i]
        seen[num] = i
    return []

# 模板：去重
def remove_duplicates(nums):
    seen = set()
    result = []
    for num in nums:
        if num not in seen:
            seen.add(num)
            result.append(num)
    return result
```

```python
# 模板：二分查找
def binary_search(nums, target):
    left, right = 0, len(nums) - 1
    
    while left <= right:
        mid = (left + right) // 2
        if nums[mid] == target:
            return mid
        elif nums[mid] < target:
            left = mid + 1
        else:
            right = mid - 1
    
    return -1

# 模板：自定义排序
def custom_sort(items):
    # 按多个条件排序
    return sorted(items, key=lambda x: (x[0], -x[1]))

# 模板：Top K
def top_k(nums, k):
    return sorted(nums, reverse=True)[:k]
    # 或使用堆：heapq.nlargest(k, nums)
```

### 组件 A：窗口函数 - 常见语法错误

```
题目：LeetCode 184 - Department Highest Salary
日期：Day 1

错误 1: ROW_NUMBER() 的 OVER() 语法
  ❌ 错误：ROW_NUMBER() OVER(PARTITION BY department)
  ✅ 正确：ROW_NUMBER() OVER(PARTITION BY department ORDER BY salary DESC)
  原因：忘记写 ORDER BY

错误 2: 窗口函数在 WHERE 中使用
  ❌ 错误：SELECT * FROM employees WHERE ROW_NUMBER() OVER(...) = 1
  ✅ 正确：需要子查询，SELECT * FROM (SELECT ..., ROW_NUMBER() OVER(...) as rn FROM employees) WHERE rn = 1
  原因：窗口函数不能在 WHERE 中直接使用

错误 3: ROW_NUMBER() vs RANK() vs DENSE_RANK()
  ❌ 错误：混淆了这三个函数的区别
  ✅ 正确：ROW_NUMBER() 连续排名，RANK() 并列排名，DENSE_RANK() 密集排名
  原因：没有理解区别

---

题目：LeetCode 177 - Nth Highest Salary
日期：Day 1

错误 4: LIMIT 的语法
  ❌ 错误：LIMIT N-1, 1（N=2 时）
  ✅ 正确：LIMIT 1 OFFSET N-1 或 使用窗口函数
  原因：LIMIT 的语法不熟练
```

---

### 组件 B：连续性问题 - 常见语法错误

```
题目：LeetCode 197 - Rising Temperature
日期：Day 2

错误 5: DATEDIFF() 的语法
  ❌ 错误：DATEDIFF(date1, date2) 
  ✅ 正确：DATEDIFF(date2, date1) 或 DATEDIFF(date1, date2) = 1（注意顺序）
  原因：DATEDIFF 的参数顺序记错了

错误 6: 自连接的 ON 条件
  ❌ 错误：ON t1.id = t2.id
  ✅ 正确：ON t1.id = t2.id AND DATEDIFF(t2.recordDate, t1.recordDate) = 1
  原因：忘记写日期条件

---

题目：LeetCode 550 - Game Play Analysis IV
日期：Day 2

错误 7: DATEDIFF() 在 CASE WHEN 中的使用
  ❌ 错误：CASE WHEN DATEDIFF(...) THEN ... END
  ✅ 正确：需要先计算，再判断
  原因：在 CASE WHEN 中直接使用 DATEDIFF 导致语法错误
```

### 组件 C：时间转换 - 常见语法错误

```
题目：LeetCode 1193 - Monthly Transactions I
日期：Day 3

错误 8: DATE_FORMAT() 的格式字符串
  ❌ 错误：DATE_FORMAT(trans_date, '%Y-%m') 
  ✅ 正确：DATE_FORMAT(trans_date, '%Y-%m')（这个是对的，但要记住格式）
  原因：格式字符串记错了

错误 9: CASE WHEN 在聚合函数中的位置
  ❌ 错误：CASE WHEN ... THEN ... END SUM(...)
  ✅ 正确：SUM(CASE WHEN ... THEN ... ELSE 0 END)
  原因：CASE WHEN 应该在 SUM 里面

错误 10: ROUND() 的语法
  ❌ 错误：ROUND(percentage, 2)
  ✅ 正确：ROUND(percentage, 2) 或 CAST(percentage AS DECIMAL(5,2))
  原因：需要指定小数位数

---

题目：LeetCode 1934 - Confirmation Rate
日期：Day 3

错误 11: LEFT JOIN 的 ON 条件
  ❌ 错误：LEFT JOIN ... ON user_id = user_id
  ✅ 正确：LEFT JOIN ... ON t1.user_id = t2.user_id
  原因：忘记加表别名

错误 12: CASE WHEN 处理 NULL
  ❌ 错误：CASE WHEN action = 'confirmed' THEN 1 END
  ✅ 正确：CASE WHEN action = 'confirmed' THEN 1 ELSE 0 END（或处理 NULL）
  原因：忘记 ELSE，导致 NULL 值
```

### 组件 D：条件聚合 - 常见语法错误

```
题目：LeetCode 1179 - Reformat Department Table
日期：Day 4

错误 13: CASE WHEN + SUM 的完整语法
  ❌ 错误：SUM(CASE WHEN month = 'Jan' THEN revenue)
  ✅ 正确：SUM(CASE WHEN month = 'Jan' THEN revenue ELSE 0 END)
  原因：忘记 ELSE 0

错误 14: GROUP BY 的语法
  ❌ 错误：GROUP BY id, name（如果有 name）
  ✅ 正确：GROUP BY id（如果只按 id 分组）
  原因：GROUP BY 的字段选择错误
```

### 组合组件 - 常见语法错误

```
题目：组合多个组件的题目
日期：Day 4-5

错误 15: 多个 CTE 之间的逗号
  ❌ 错误：WITH t1 AS (...) WITH t2 AS (...)
  ✅ 正确：WITH t1 AS (...), t2 AS (...)
  原因：多个 CTE 之间应该用逗号，不是第二个 WITH

错误 16: 子查询的别名
  ❌ 错误：SELECT * FROM (SELECT ...)
  ✅ 正确：SELECT * FROM (SELECT ...) AS t
  原因：子查询必须要有别名（MySQL 可能不需要，但最好加上）

错误 17: 窗口函数 + 聚合函数的组合
  ❌ 错误：在聚合函数外面再用窗口函数
  ✅ 正确：先窗口函数，再聚合，或者用子查询
  原因：语法顺序错误
```
