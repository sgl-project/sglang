# 学习焦虑缓解指南：技能优先级与学习策略

## 🎯 核心观点：没有人什么都会

### 现实情况

```
❌ 错误认知：
  "找工作的人 LeetCode、Pandas、LLM 什么都会"
  
✅ 实际情况：
  - 每个人都有自己的"技能树"
  - 面试官更看重"深度"而不是"广度"
  - 80% 的岗位只需要 20% 的核心技能
```

### 真实案例

**AI Infra 工程师（SGLang 团队）**：
- ✅ 深度：调度系统、内存管理、算子优化
- ✅ 广度：了解 LLM 推理流程
- ❌ 不需要：Pandas 精通、SQL 复杂查询

**Data Engineer**：
- ✅ 深度：SQL 业务逻辑、数据管道设计
- ✅ 广度：了解数据仓库架构
- ❌ 不需要：LeetCode Hard、LLM 训练细节

**LLM 研究员**：
- ✅ 深度：模型架构、训练优化
- ✅ 广度：了解推理系统
- ❌ 不需要：Pandas 复杂操作、SQL 优化

## 💡 学习策略：不要"完美主义"

### 策略 1：够用就行（Good Enough）

```
❌ 完美主义：
  "我要把 LeetCode 刷 500 道"
  "我要把 Pandas 所有 API 都记住"
  "我要把 LLM 所有细节都理解"

✅ 够用策略：
  "LeetCode：刷 30-50 道经典题，覆盖常见模式"
  "Pandas：掌握 10-15 个常用操作，其他查文档"
  "LLM：理解推理流程，不需要训练细节"
```

### 策略 2：按需学习（Just-In-Time Learning）

```
❌ 提前学习：
  "万一面试问到 Pandas，我先全部学完"
  "万一面试问到 LLM，我先全部学完"

✅ 按需学习：
  "面试前 1-2 天，快速过一遍 Pandas 常用操作"
  "面试前 1-2 天，快速过一遍 LLM 推理流程"
  "不会的当场查文档，展示学习能力"
```

### 策略 3：展示学习能力（比"什么都会"更重要）

```
面试官更看重：
  ✅ "我不会，但我能快速学习"
  ✅ "我不会，但我能查文档解决问题"
  ✅ "我不会，但我能分析问题并给出思路"

而不是：
  ❌ "我什么都会"（不现实，也不可信）
```

## 🎯 面试时的"不会"策略

### 当面试官问到你不熟悉的内容

**错误回答**：
```
❌ "我不会"（直接放弃）
❌ "我没学过"（显得不主动）
❌ "这个不重要"（显得不谦虚）
```

**正确回答**：
```
✅ "这个问题我不太熟悉，但我可以尝试分析一下..."
✅ "我不确定，但我可以查一下文档..."
✅ "这个我没做过，但根据我的理解，应该是..."
```

### 展示学习能力

```
面试官问："你会用 Pandas 处理时间序列吗？"

❌ 错误回答：
  "我不会，我没学过"

✅ 正确回答：
  "我平时主要用 SQL，Pandas 用得不多。
   但根据我的理解，时间序列处理通常需要：
   - 时间索引设置（pd.to_datetime）
   - 重采样（resample）
   - 滑动窗口（rolling）
   
   如果遇到具体问题，我会查文档快速上手。
   我的优势是能快速将业务需求转换为代码逻辑。"
```
