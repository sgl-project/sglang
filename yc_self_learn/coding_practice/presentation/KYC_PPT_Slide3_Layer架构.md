## 设计说明

### 颜色方案（按Layer分层配色）

- **Layer 1 (输入与预处理层)**: 
  - 背景: 浅蓝色 #e0f2fe
  - 边框: 深蓝色 #1e3a8a
  - 说明: 清爽的蓝色，代表数据输入

- **Layer 2 (编排与限流层)**: 
  - 背景: 浅绿色 #dcfce7
  - 边框: 深绿色 #166534
  - 说明: 绿色代表控制和协调

- **Layer 3 (推理层)**: 
  - 背景: 橙色 #f97316 (突出显示)
  - 边框: 深橙色 #ea580c
  - 说明: 橙色突出显示，代表核心推理层

- **Layer 4 (验证层)**: 
  - 背景: 浅紫色 #f3e8ff
  - 边框: 深紫色 #7e22ce
  - 说明: 紫色代表验证和确认

- **Layer 5 (输出层)**: 
  - 背景: 浅青色 #cffafe
  - 边框: 深青色 #155e75
  - 说明: 青色代表结果输出

### 布局特点

1. **清晰的Layer分隔**: 每个Layer用不同颜色的背景框区分
2. **垂直流向**: 数据从上到下流动，符合Pipeline概念
3. **Layer标题**: 每层有清晰的标题栏，便于讲解时定位
4. **核心层突出**: Layer 3用橙色突出显示，强调核心推理层
5. **并列组件**: Layer 1和Layer 4中的组件并列显示，表示可以并行处理

### 讲解建议

**按Layer顺序讲解（每层1.5-2分钟）**:

1. **Layer 1** (1.5分钟): 介绍输入格式和预处理步骤
2. **Layer 2** (1.5分钟): 讲解限流和并发控制的重要性
3. **Layer 3** (2分钟): **重点讲解** - 多模态LLM推理，这是核心
4. **Layer 4** (2分钟): 讲解验证和规则引擎的作用
5. **Layer 5** (1分钟): 介绍输出格式和决策结果

**讲解话术示例**:
- "让我们从第一层开始..."
- "进入第二层，我们看到..."
- "这是最核心的第三层，多模态LLM推理..."
- "经过推理后，数据进入第四层验证..."
- "最后，在第五层我们生成结构化输出..."

### 动画建议

1. **Layer顺序出现**: 从上到下，逐层淡入
2. **组件内部动画**: Layer 1和Layer 4中的并列组件可以同时出现
3. **箭头跟随**: 每层出现后，箭头跟随出现
4. **突出显示**: Layer 3出现时，可以用放大或闪烁效果

## 与原始架构图的区别

1. **分层结构**: 将组件按功能分为5个清晰的Layer
2. **视觉区分**: 每个Layer用不同颜色背景，便于区分
3. **讲解友好**: 可以逐层讲解，逻辑清晰
4. **核心突出**: Layer 3用特殊颜色突出显示
5. **总结区域**: 底部有Layer总结和延迟统计
