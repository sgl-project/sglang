## 核心问题

**问题**：如何计算 A/B Test 的 cost？

**答案**：A/B Test 的成本包括**时间成本**、**资源成本**、**人力成本**三部分。通过详细计算每部分成本，可以估算总成本，并优化实验设计。

---

## 成本组成

### 1. 成本分类

**A/B Test 的成本包括三部分**：

| 成本类型 | 说明 | 计算公式 |
|---------|------|---------|
| **时间成本** | 实验时长带来的机会成本 | 实验时长 × 单位时间成本 |
| **资源成本** | 基础设施、API 调用等资源成本 | 样本量 × 单位资源成本 |
| **人力成本** | 设计、分析、维护的人力投入 | 人力投入 × 单位人力成本 |

**总成本** = 时间成本 + 资源成本 + 人力成本

---

## 详细计算方法

### 1. 时间成本（Time Cost）

**定义**：实验时长带来的机会成本（延迟决策、延迟优化）

**计算公式**：
```
时间成本 = 实验时长（周）× 单位时间成本（$/周）
```

**单位时间成本**：
- **机会成本**：延迟优化带来的损失（如流失率、转化率）
- **业务影响**：实验期间无法全量发布带来的影响

**KYC 示例**：
```
场景：验证 p95 阈值（实验时长 2 周）

时间成本计算：
  - 实验时长：2 周
  - 单位时间成本：
    - 如果 p95 = 20s（实验组 B），流失率 = 12%
    - 如果 p95 = 10s（实验组 A），流失率 = 5%
    - 机会成本 = (12% - 5%) × 10,000 用户 × $100/用户 = $70,000/周
  - 时间成本 = 2 周 × $70,000/周 = $140,000

说明：这是"如果实验组 B 更差，延迟优化的成本"
```

---

### 2. 资源成本（Resource Cost）

**定义**：基础设施、API 调用等资源成本

**计算公式**：
```
资源成本 = 样本量 × 单位资源成本
```

**资源成本组成**：

| 资源类型 | 说明 | 计算公式 |
|---------|------|---------|
| **API 调用成本** | 调用外部 API 的成本 | 请求数 × API 单价 |
| **基础设施成本** | 服务器、存储等成本 | 实验时长 × 基础设施单价 |
| **数据存储成本** | 存储实验数据的成本 | 数据量 × 存储单价 |

**KYC 示例**：
```
场景：验证重试次数（3 次 vs 5 次）

资源成本计算：
  - 样本量：每组 50,000 请求（共 100,000 请求）
  - 实验时长：1 周
  - API 调用成本：
    - 实验组 A（3 次重试）：平均 1.2 次调用/请求
    - 实验组 B（5 次重试）：平均 1.5 次调用/请求
    - API 单价：$0.0015 / 调用
    - 实验组 A 成本 = 50,000 × 1.2 × $0.0015 = $90
    - 实验组 B 成本 = 50,000 × 1.5 × $0.0015 = $112.5
    - API 调用总成本 = $90 + $112.5 = $202.5
  - 基础设施成本：
    - 服务器成本：$100/周
    - 存储成本：$10/周
    - 基础设施总成本 = $110/周 × 1 周 = $110
  - 资源总成本 = $202.5 + $110 = $312.5
```

---

### 3. 人力成本（Human Cost）

**定义**：设计、分析、维护的人力投入

**计算公式**：
```
人力成本 = 人力投入（人周）× 单位人力成本（$/人周）
```

**人力投入组成**：

| 阶段 | 说明 | 时间投入 |
|------|------|---------|
| **设计阶段** | 设计实验、定义假设、确定指标 | 0.1-0.2 人周 |
| **实施阶段** | 配置实验、部署代码、监控 | 0.1-0.2 人周 |
| **分析阶段** | 数据分析、统计显著性、报告 | 0.1-0.2 人周 |
| **维护阶段** | 监控实验、处理异常 | 0.05-0.1 人周 |

**KYC 示例**：
```
场景：验证 p95 阈值（实验时长 2 周）

人力成本计算：
  - 设计阶段：0.2 人周（设计实验、定义假设）
  - 实施阶段：0.1 人周（配置实验、部署代码）
  - 分析阶段：0.2 人周（数据分析、统计显著性）
  - 维护阶段：0.1 人周（监控实验、处理异常）
  - 总人力投入 = 0.2 + 0.1 + 0.2 + 0.1 = 0.6 人周
  - 单位人力成本 = $5,000 / 人周（假设）
  - 人力成本 = 0.6 人周 × $5,000 / 人周 = $3,000
```

---

## 总成本计算

### 1. 总成本公式

**总成本** = 时间成本 + 资源成本 + 人力成本

**KYC 示例**：
```
场景：验证 p95 阈值（实验时长 2 周）

总成本计算：
  - 时间成本：$140,000（机会成本）
  - 资源成本：$312.5（API 调用 + 基础设施）
  - 人力成本：$3,000（设计 + 分析 + 维护）
  - 总成本 = $140,000 + $312.5 + $3,000 = $143,312.5

说明：
  - 时间成本占主导（97.7%）
  - 资源成本和人力成本相对较小（2.3%）
```

---

### 2. 成本优化策略

**优化目标**：在保证统计显著性的前提下，最小化总成本

**优化策略**：

| 策略 | 说明 | 效果 |
|------|------|------|
| **减少实验时长** | 通过增加样本量，缩短实验时长 | 减少时间成本 |
| **减少样本量** | 通过延长实验时长，减少样本量 | 减少资源成本 |
| **优化人力投入** | 自动化设计、分析、维护流程 | 减少人力成本 |
| **选择合适的时间窗口** | 避开高峰期，选择低流量时段 | 减少资源成本 |

**KYC 示例**：
```
场景：验证 p95 阈值

优化前：
  - 实验时长：2 周
  - 样本量：每组 10,000 用户
  - 总成本：$143,312.5

优化后（增加样本量，缩短实验时长）：
  - 实验时长：1 周（样本量增加到 20,000 用户/组）
  - 样本量：每组 20,000 用户
  - 时间成本：$70,000（减少 50%）
  - 资源成本：$625（增加 100%，但绝对值小）
  - 人力成本：$3,000（不变）
  - 总成本：$73,625（减少 48.6%）

结论：通过增加样本量，缩短实验时长，可以显著减少总成本
```

### 2. 成本敏感性分析

**分析不同参数对总成本的影响**：

| 参数 | 基准值 | 变化范围 | 对总成本的影响 |
|------|--------|---------|---------------|
| **实验时长** | 2 周 | 1-4 周 | 线性影响（时间成本占主导） |
| **样本量** | 10,000 用户/组 | 5,000-20,000 用户/组 | 线性影响（资源成本） |
| **API 单价** | $0.0015/请求 | $0.001-0.002/请求 | 线性影响（资源成本） |
| **人力成本** | $5,000/人周 | $3,000-7,000/人周 | 线性影响（人力成本） |

**KYC 示例**：
```
敏感性分析：

1. 实验时长从 2 周减少到 1 周：
   - 时间成本：$140,000 → $70,000（-50%）
   - 总成本：$143,371 → $73,371（-48.8%）

2. 样本量从 10,000 增加到 20,000：
   - 资源成本：$371 → $742（+100%）
   - 但可以缩短实验时长，总成本可能减少

3. API 单价从 $0.0015 增加到 $0.002：
   - 资源成本：$371 → $494（+33%）
   - 总成本：$143,371 → $143,494（+0.09%）

结论：实验时长对总成本影响最大，应该优先优化
```

## 业界最佳实践

### 1. 成本控制策略

**业界实践**：

| 公司 | 成本控制策略 | 效果 |
|------|------------|------|
| **Google** | 大量 A/B Test，但严格控制实验时长（通常 1-2 周） | 时间成本可控 |
| **Amazon** | 定期 A/B Test，但优化样本量（通常 5-10% 流量） | 资源成本可控 |
| **Netflix** | 自动化 A/B Test 流程，减少人力投入 | 人力成本可控 |
| **Facebook** | 使用内部 A/B Test 平台，降低基础设施成本 | 资源成本可控 |

**共同点**：
1. **严格控制实验时长**：通常 1-2 周，不超过 4 周
2. **优化样本量**：通常 5-10% 流量，不超过 20%
3. **自动化流程**：减少人力投入，提高效率
4. **使用内部平台**：降低基础设施成本

---

### 2. ROI 计算

**ROI（Return on Investment）**：A/B Test 的收益 / 成本

**计算公式**：
```
ROI = (收益 - 成本) / 成本 × 100%
```

**KYC 示例**：
```
场景：验证 p95 阈值

成本：$143,371
收益：
  - 如果实验组 B（p95=20s）更差，避免全量发布带来的损失
  - 流失率差异：12% - 5% = 7%
  - 如果全量发布实验组 B，损失 = 7% × 100,000 用户 × $100/用户 = $700,000
  - 收益 = $700,000（避免的损失）

ROI = ($700,000 - $143,371) / $143,371 × 100% = 388%

结论：A/B Test 的 ROI 很高，值得进行
```

## 总结

### 核心计算方法

1. **时间成本** = 实验时长 × 单位时间成本
2. **资源成本** = 样本量 × 单位资源成本
3. **人力成本** = 人力投入 × 单位人力成本
4. **总成本** = 时间成本 + 资源成本 + 人力成本

### 成本优化策略

1. **减少实验时长**：通过增加样本量，缩短实验时长
2. **优化样本量**：在保证统计显著性的前提下，最小化样本量
3. **自动化流程**：减少人力投入，提高效率
4. **选择合适的时间窗口**：避开高峰期，选择低流量时段

### 业界最佳实践

- **严格控制实验时长**：通常 1-2 周，不超过 4 周
- **优化样本量**：通常 5-10% 流量，不超过 20%
- **自动化流程**：减少人力投入，提高效率
- **计算 ROI**：确保 A/B Test 的收益大于成本
