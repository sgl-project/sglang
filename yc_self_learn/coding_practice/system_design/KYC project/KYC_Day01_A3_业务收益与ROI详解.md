## Definition（定义）

**L1 业务收益（ROI）**：**系统为业务创造的实际价值**（省钱、省时、降低风险）。

**核心问题**：**"系统创造价值了吗？"**

**谁关心**：
- ✅ **业务团队/PM**：证明系统的价值，争取资源
- ✅ **财务部门**：成本分析，预算规划
- ✅ **决策层**：是否继续投资，是否扩大规模

**与 L0 的区别**：
- **L0**：系统是否正常工作（技术指标）
- **L1**：系统是否创造价值（业务指标）

## 🎯 核心指标详解

### 指标 1：每单节省的人审分钟数

**定义**：**自动化处理相比人工审核节省的时间**。

**KYC 项目示例**：

#### 基线（人工审核）

```
人工审核一张 ID 文档：
- 查看文档：1-2 分钟
- 验证信息：2-3 分钟
- 录入系统：1-2 分钟
- 复核：1-2 分钟
总计：5-10 分钟/单
```

#### 当前值（AI 自动化）

```
AI 处理一张 ID 文档：
- 图片预处理：0.5 秒
- LLM 推理：2-3 秒
- 验证和规则检查：0.5-1 秒
总计：3-5 秒/单
```

#### 节省计算

```
节省时间 = 人工时间 - AI 时间
         = 5 分钟/单 - 5 秒/单
         = 4 分 55 秒/单
         ≈ 5 分钟/单

节省效率 = (5 分钟 - 5 秒) / 5 分钟
         = 99.8%
         ≈ > 99% 的时间节省
```

#### ROI 计算

**假设**：每月处理 1000 单

```
每月节省时间 = 5 分钟/单 × 1000 单/月
            = 5000 分钟/月
            = 83.3 小时/月
            ≈ 83 小时/月

每年节省时间 = 83 小时/月 × 12 月
            = 996 小时/年
            ≈ 1000 小时/年

人力成本节省（假设 $50/小时）：
每月节省 = 83 小时 × $50/小时 = $4,150/月
每年节省 = 1000 小时 × $50/小时 = $50,000/年
```

**为什么重要**：
- ✅ **直接证明价值**：业务团队最关心的指标
- ✅ **可量化**：可以计算人力成本节省
- ✅ **易于理解**：非技术人员也能理解

### 指标 2：错误拦截率带来的风险降低

**定义**：**系统自动拦截的潜在风险案例数**。

**KYC 项目示例**：

#### 关键拦截场景（基于 `src/rules.py` 的 fraud markers）

```python
# src/rules.py 中的 fraud markers
fraud_markers = [
    "expiry:expired",  # 过期文档拦截
    "missing_critical:full_name,date_of_birth",  # 关键字段缺失拦截
    "low_confidence_critical:document_number",  # 低置信度关键字段拦截
    "quality:too_blurry",  # 图片质量太差拦截
    "format:unsupported"  # 格式不支持拦截
]
```

#### 基线（无自动化拦截）

```
人工审核：
- 人工审核员可能漏检：10-20%
- 无自动化拦截：0%
- 潜在风险案例通过：100-200 件/月
```

#### 当前值（AI 自动化拦截）

```
AI 自动化拦截：
- 拦截率：15-20%（PoV 阶段）
- 拦截的案例数：150-200 件/月
- 风险降低：避免的潜在合规风险
```

#### 风险降低计算

**假设**：每月处理 1000 单，拦截率 15%

```
拦截案例数 = 1000 单/月 × 15%
          = 150 件/月

避免的风险：
- 合规风险：避免 150 件不合规案例通过
- 法律风险：降低潜在的法律诉讼风险
- 品牌风险：避免因不合规案例导致的品牌损害
```

**风险价值估算**：

```
假设每件不合规案例的潜在损失：
- 合规罚款：$1,000-10,000/件
- 法律诉讼：$10,000-100,000/件
- 品牌损害：难以量化，但影响长期

保守估算（每件 $5,000）：
每月避免损失 = 150 件 × $5,000/件
            = $750,000/月

每年避免损失 = $750,000/月 × 12 月
            = $9,000,000/年
```

**为什么重要**：
- ✅ **降低合规风险**：KYC 是强监管领域，合规风险高
- ✅ **减少人工漏检**：AI 可以 24/7 工作，不会疲劳
- ✅ **降低法律风险**：避免潜在的法律诉讼

### 指标 3：吞吐提升带来的成本节省

**定义**：**自动化处理相比人工处理的成本差异**。

**KYC 项目示例**：

#### 成本分解

**人工审核成本**：
```
人工审核员成本：
- 时薪：$50/小时（假设）
- 每单时间：5-10 分钟/单
- 每单成本：$5-10/单
平均：$7.5/单
```

**AI 自动化成本**：
```
Fireworks API 调用成本：
- Qwen2.5-VL-32B 模型：$0.001-0.002 / request
- 图片预处理：$0（本地处理）
- 验证和规则检查：$0（本地处理）
平均：$0.0015/单
```

#### 成本对比

```
人工成本：$7.5/单
AI 成本：$0.0015/单
节省：$7.4985/单
节省比例：99.98%
```

#### ROI 计算

**假设**：每月处理 1000 单

```
每月成本对比：
人工成本 = $7.5/单 × 1000 单/月 = $7,500/月
AI 成本 = $0.0015/单 × 1000 单/月 = $1.5/月
节省 = $7,500 - $1.5 = $7,498.5/月

每年成本对比：
人工成本 = $7,500/月 × 12 月 = $90,000/年
AI 成本 = $1.5/月 × 12 月 = $18/年
节省 = $90,000 - $18 = $89,982/年

ROI 倍数 = $90,000 / $18 = 5,000x 成本降低
```

**为什么重要**：
- ✅ **直接的成本节省**：财务部门最关心的指标
- ✅ **可量化 ROI**：可以计算投资回报率
- ✅ **规模化优势**：处理量越大，节省越多

### 指标 4：自动化率

**定义**：**无需人工介入的请求数 / 总请求数**。

**KYC 项目示例**：

#### 自动化判断标准（基于 `src/rules.py`）

```python
# 自动化判断条件
def is_automated(result: dict) -> bool:
    """判断是否自动化处理（无需人工 review）"""
    
    # 1. 所有关键字段提取成功
    required_fields = [
        "full_name",
        "date_of_birth",
        "document_number",
        "expiry_date",
        "issuing_country"
    ]
    if not all(field in result for field in required_fields):
        return False
    
    # 2. 置信度 > 阈值（如 > 0.85）
    if result.get("confidence", 0) < 0.85:
        return False
    
    # 3. 通过确定性规则检查
    if result.get("needs_review", False):
        return False
    
    # 4. 无 fraud markers
    if result.get("fraud_markers"):
        return False
    
    return True
```

#### 目标 vs 当前值

```
目标：80%（20% 需要人工 review）
当前值：60-70%（PoV 阶段）
差距：10-20%（需要优化）
```

#### 自动化率计算

**假设**：每月处理 1000 单，自动化率 65%

```
自动化处理数 = 1000 单/月 × 65% = 650 单/月
人工 review 数 = 1000 单/月 × 35% = 350 单/月

如果达到目标（80%）：
自动化处理数 = 1000 单/月 × 80% = 800 单/月
人工 review 数 = 1000 单/月 × 20% = 200 单/月

额外节省 = (800 - 650) 单/月 × 5 分钟/单
        = 150 单/月 × 5 分钟/单
        = 750 分钟/月
        = 12.5 小时/月
```

**为什么重要**：
- ✅ **反映系统成熟度**：自动化率越高，系统越成熟
- ✅ **直接关联人力成本**：自动化率越高，人力成本越低
- ✅ **业务效率指标**：业务团队关心的效率指标

## 🔍 如何计算 L1 指标（基于 `_summary.json`）

### 从 `_summary.json` 提取 L1 指标

```python
# 计算 L1 指标
def calculate_l1_metrics(summary_path: Path) -> dict:
    """从 _summary.json 计算 L1 业务收益指标"""
    
    with open(summary_path) as f:
        summary = json.load(f)
    
    results = summary["results"]
    total_files = len(results)
    
    # 1. 计算自动化率
    automated_count = sum(
        1 for r in results
        if is_automated(r)  # 见上面的 is_automated() 函数
    )
    automation_rate = automated_count / total_files if total_files > 0 else 0
    
    # 2. 计算平均处理时间（用于计算时间节省）
    latencies = [r.get("latency_ms", 0) for r in results if r.get("status") == "success"]
    avg_latency_ms = sum(latencies) / len(latencies) if latencies else 0
    avg_latency_seconds = avg_latency_ms / 1000
    
    # 3. 计算风险拦截数（fraud markers）
    fraud_count = sum(
        1 for r in results
        if r.get("fraud_markers")
    )
    risk_interception_rate = fraud_count / total_files if total_files > 0 else 0
    
    # 4. 计算成本（基于 API 调用）
    total_cost = sum(r.get("cost_usd", 0) for r in results)
    avg_cost_per_file = total_cost / total_files if total_files > 0 else 0
    
    return {
        "automation_rate": automation_rate,
        "avg_latency_seconds": avg_latency_seconds,
        "risk_interception_rate": risk_interception_rate,
        "avg_cost_per_file": avg_cost_per_file,
        "total_cost": total_cost,
        "automated_count": automated_count,
        "fraud_count": fraud_count
    }
```

### 优化策略

**提升自动化率**（从 65% 到 80%）：
- ✅ **优化置信度阈值**：调整 `confidence_threshold`，平衡准确性和自动化率
- ✅ **优化 Schema 验证**：放宽非关键字段的验证要求
- ✅ **优化业务规则**：减少不必要的 `needs_review` 标记

**降低风险拦截率**（从 15% 到 20-25%）：
- ✅ **增强 fraud detection**：添加更多 fraud markers
- ✅ **优化规则引擎**：提高规则覆盖率和准确性
- ✅ **持续学习**：根据历史数据优化规则
