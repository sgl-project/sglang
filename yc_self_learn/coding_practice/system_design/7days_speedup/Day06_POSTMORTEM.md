# Day 6｜Postmortem 模板：把"出事"变成组织学习

## 为什么要写 Postmortem

Postmortem（复盘）不是为了追责，而是为了：
1. **快速止血**：记录事故处理过程
2. **防止复发**：找出根因，制定行动项
3. **组织学习**：积累经验，改进系统

## Postmortem 原则

### Blameless Culture（无责文化）

- **不是为了追责**：找出问题，不是找出责任人
- **关注系统和流程**：问题通常来自系统设计，不是个人错误
- **鼓励公开分享**：只有公开分享，才能组织学习

### 关键要素

1. **及时性**：事故后 48 小时内完成
2. **完整性**：包含所有关键信息
3. **可行动性**：每个行动项都有负责人和 DDL
4. **可追踪性**：跟踪行动项完成情况
