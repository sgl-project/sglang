# Day 3｜回归与门禁：把 AI 系统变成"可回测工程"

## 为什么要练

你现在最容易被误判的点就是"能交付，但改动会不会把系统搞坏？"回归集 + 指标门禁是最硬的 senior 信号。

## 目的

让每次 prompt/模型/validator 改动都有 before/after 可证据化。

## 目标

建立最小黄金集 + 通过阈值（release gate）。

---

## Golden Set（黄金测试集）

### 构建策略

**目标规模**：50-200 条测试用例

**覆盖场景**：
- [ ] **正常场景**（20%）：清晰、标准格式
- [ ] **边界场景**（30%）：模糊、遮挡、低质量
- [ ] **异常场景**（30%）：版式变化、多页、复杂布局
- [ ] **长尾场景**（20%）：罕见格式、特殊字符、极端情况

### Golden Set 分类

#### 1. Hard Cases（困难案例）

**模糊/遮挡**：
- [ ] 图片模糊（`____` 个案例）
- [ ] 部分遮挡（`____` 个案例）
- [ ] 低分辨率（`____` 个案例）

**版式变化**：
- [ ] 非常规布局（`____` 个案例）
- [ ] 多栏布局（`____` 个案例）
- [ ] 表格密集（`____` 个案例）

**长尾情况**：
- [ ] 特殊字符/编码（`____` 个案例）
- [ ] 多语言混排（`____` 个案例）
- [ ] 极端长度（`____` 个案例）

#### 2. Critical Cases（关键案例）

**业务关键字段**：
- [ ] 必须提取成功的字段（`____` 个案例）
- [ ] 对业务影响大的字段（`____` 个案例）

### Golden Set 维护

- **版本管理**：Golden Set 版本化，与模型/prompt 版本绑定
- **定期审查**：每月回顾，补充新发现的 edge cases
- **自动化**：CI/CD 自动运行回归测试

## 门禁指标（Release Gate）

### 核心指标

#### 1. Schema Pass Rate（Schema 通过率）

- **定义**：Schema 验证通过的请求数 / 总请求数
- **阈值**：`> ____%`
- **当前值**：`____%`
- **门禁规则**：如果 < 阈值，**禁止发布**

#### 2. 字段级准确率（Field-level Accuracy）

- **定义**：正确提取的字段数 / 总字段数
- **阈值**：`> ____%`
- **当前值**：`____%`
- **门禁规则**：如果 < 阈值，**禁止发布**

#### 3. 字段级一致性（Field-level Consistency）

- **定义**：相同输入多次提取结果一致的字段数 / 总字段数
- **阈值**：`> ____%`
- **当前值**：`____%`
- **门禁规则**：如果 < 阈值，**禁止发布**

#### 4. Fallback 比例（Fallback Rate）

- **定义**：触发降级的请求数 / 总请求数
- **阈值**：`< ____%`
- **当前值**：`____%`
- **门禁规则**：如果 > 阈值，**禁止发布**

#### 5. 成本上限（Cost Cap）

- **定义**：tokens / request 或 $ / request
- **阈值**：`< ____ tokens/request` 或 `$ < ____ / request`
- **当前值**：`____ tokens/request` 或 `$____ / request`
- **门禁规则**：如果 > 阈值，**禁止发布**

## 回归测试流程

### 发布前必跑

```
┌─────────────────┐
│   代码变更       │
│ (Prompt/Model/  │
│  Validator)     │
└────────┬────────┘
         │
         ▼
┌─────────────────┐      ┌─────────────────┐
│  运行 Golden    │ ────▶│   计算指标      │
│     Set         │      │ (Accuracy/      │
│                 │      │  Cost/etc.)     │
└─────────────────┘      └────────┬────────┘
                                  │
                    ┌─────────────┴─────────────┐
                    │                           │
                    ▼                           ▼
         ┌─────────────────┐      ┌─────────────────┐
         │   通过门禁?     │ Yes  │   可以发布      │
         │  (All metrics   │─────▶│                 │
         │   meet gates)   │      │                 │
         └─────────────────┘      └─────────────────┘
                    │
                  No
                    │
                    ▼
         ┌─────────────────┐
         │   禁止发布      │
         │   修复后再测    │
         └─────────────────┘
```

### 回归测试执行步骤

1. **准备环境**
   - [ ] 部署新版本到测试环境
   - [ ] 准备 Golden Set 数据

2. **运行测试**
   - [ ] 批量执行 Golden Set
   - [ ] 收集结果（输出、延迟、成本）

3. **计算指标**
   - [ ] Schema Pass Rate
   - [ ] 字段级准确率
   - [ ] 一致性
   - [ ] Fallback 比例
   - [ ] 成本

4. **对比基准**
   - [ ] 与上一版本对比（Before/After）
   - [ ] 识别退化（Regression）

5. **门禁判断**
   - [ ] 所有指标是否满足阈值？
   - [ ] 是否有明显退化？

6. **决策**
   - [ ] 通过 → 允许发布
   - [ ] 不通过 → 修复后再测
