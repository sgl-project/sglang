# HydroSense ML Tech Block

## 1. ML 架构概览

HydroSense 采用**端云协同**的 ML 架构：端侧轻量模型（实时推理）+ 云端重模型（离线训练与回放验证）。

```
Edge ML (STM32) → Cloud ML (Training/Validation) → Model Deployment → Edge Update
```

## 2. 端侧 ML（Edge ML）

### 2.1 端侧模型需求

**约束**：
- **模型大小**：< 100KB（Flash 限制）
- **推理延迟**：< 100ms（实时性要求）
- **功耗**：< 10mA（低功耗要求）
- **精度**：满足基本预测需求（可接受精度损失）

**应用场景**：
- 降雨预测（短期，如未来 1 小时）
- 异常检测（实时异常识别）
- 数据质量评估（缺失/异常标记）

### 2.2 模型选择

**轻量 CNN 模型**：
- **架构**：1D CNN（时序数据）
- **输入**：滑动窗口（如最近 60 个采样点）
- **输出**：预测值 + 置信度
- **参数量**：~10K（量化后 < 100KB）

**模型结构示例**：
```
Input (60, 4) → Conv1D(16) → ReLU → MaxPool → 
Conv1D(32) → ReLU → MaxPool → 
FC(64) → ReLU → FC(1) → Output
```

### 2.3 模型优化

**量化**：
- INT8 量化（FP32 → INT8）
- 精度损失 < 5%
- 模型大小减少 4 倍

**剪枝**：
- 结构化剪枝（移除不重要的通道）
- 模型大小减少 2-3 倍
- 精度损失 < 3%

**蒸馏**：
- 从云端大模型蒸馏到端侧小模型
- 保持关键特征提取能力

### 2.4 端侧推理流程

**推理流程**：
```
传感器数据 → 滑动窗口构建 → 特征提取 → 
CNN 推理 → 后处理 → 预测结果
```

**滑动窗口**：
- 窗口大小：60 个采样点（如 1 小时数据，1 分钟采样）
- 滑动步长：1 个采样点（每次推理更新）

**后处理**：
- 平滑处理（移动平均）
- 阈值过滤（置信度 < 阈值则使用规则预测）
- 结果缓存（避免重复计算）

### 2.5 模型/规则双轨

**策略**：
- **规则兜底**：可解释、鲁棒（如阈值规则）
- **CNN 提升**：提升预测能力（可在云端回放验证）

**切换逻辑**：
- CNN 置信度 > 阈值 → 使用 CNN 预测
- CNN 置信度 ≤ 阈值 → 使用规则预测
- CNN 推理失败 → 降级到规则预测

## 3. 云端 ML（Cloud ML）

### 3.1 训练数据准备

**数据源**：
- 历史传感器数据（时序数据）
- 标签数据（人工标注/规则生成）
- 设备元数据（设备类型/位置/环境）

**数据预处理**：
- 数据清洗（去噪/去异常）
- 特征工程（统计特征/频域特征/时域特征）
- 数据增强（时间扭曲/噪声注入）

**训练数据集**：
- 训练集：70%
- 验证集：15%
- 测试集：15%

### 3.2 模型训练

**模型架构**：
- **时序模型**：LSTM/GRU/Transformer（长期依赖）
- **CNN 模型**：1D/2D CNN（局部特征）
- **混合模型**：CNN + LSTM（局部 + 长期）

**训练策略**：
- 超参数调优（网格搜索/贝叶斯优化）
- 早停（Early Stopping）
- 学习率调度（Cosine Annealing）

**评估指标**：
- **回归任务**：MAE、RMSE、R²
- **分类任务**：Accuracy、Precision、Recall、F1
- **时序任务**：MAPE、SMAPE

### 3.3 模型验证与回放

**回放验证**：
- 使用历史数据回放端侧模型预测
- 对比端侧预测与云端重模型预测
- 评估端侧模型性能（精度/延迟/功耗）

**A/B 测试**：
- 新旧模型对比测试
- 关键指标对比（精度/延迟/资源消耗）
- 决定是否部署新模型

### 3.4 模型部署

**部署流程**：
1. 模型训练完成
2. 模型验证通过
3. 模型量化/剪枝优化
4. 模型打包（ONNX/TFLite）
5. 模型上传（对象存储）
6. OTA 下发到设备

**版本管理**：
- 模型版本号（如 v1.0.0）
- 模型元数据（精度/大小/延迟）
- 模型回滚（支持回滚到旧版本）

## 4. 特征工程

### 4.1 时域特征

**统计特征**：
- mean, std, min, max
- median, percentile
- skewness, kurtosis

**变化特征**：
- 一阶差分（rate of change）
- 二阶差分（acceleration）
- 变化率（percentage change）

### 4.2 频域特征

**FFT 特征**：
- 功率谱（power spectrum）
- 主频（dominant frequency）
- 频带能量（band energy）

**小波特征**：
- 小波系数（wavelet coefficients）
- 小波能量（wavelet energy）

### 4.3 领域特征

**雨量特征**：
- 瞬时雨强（rainfall rate）
- 累计雨量（accumulated rainfall）
- 雨强变化率（rainfall rate change）

**环境特征**：
- 温度/湿度相关性
- 风速影响
- 季节性特征

## 5. 异常检测

### 5.1 端侧异常检测

**方法**：
- **统计方法**：3-sigma 规则
- **滑动窗口**：窗口内异常检测
- **CNN 异常检测**：端侧轻量模型

**异常类型**：
- **传感器故障**：数据缺失/异常值
- **环境干扰**：振动/电磁干扰
- **设备异常**：电池低/通信异常

### 5.2 云端异常检测

**方法**：
- **LSTM Autoencoder**：重构误差检测异常
- **Isolation Forest**：孤立森林
- **One-Class SVM**：单类支持向量机

**异常类型**：
- **传感器漂移**：长期趋势异常
- **站点异常**：站点级异常（多设备）
- **模式异常**：异常模式识别

## 6. 模型更新策略

### 6.1 更新触发条件

**条件**：
- 模型精度下降（验证集精度 < 阈值）
- 新数据积累（如 1 个月新数据）
- 业务需求变化（新场景/新需求）

### 6.2 更新流程

**流程**：
1. 新模型训练
2. 模型验证（A/B 测试）
3. 灰度发布（1% → 10% → 50% → 100%）
4. 监控模型性能（精度/延迟/资源消耗）
5. 全量发布或回滚

### 6.3 模型版本管理

**版本策略**：
- 语义化版本（Major.Minor.Patch）
- 版本兼容性（向后兼容）
- 版本回滚（支持回滚到旧版本）

## 7. ML 可观测性

### 7.1 模型性能监控

**监控指标**：
- **精度指标**：Accuracy、Precision、Recall、F1
- **延迟指标**：推理延迟（P50/P99）
- **资源指标**：内存占用、CPU 占用、功耗

### 7.2 数据质量监控

**监控指标**：
- 数据分布偏移（Distribution Shift）
- 特征漂移（Feature Drift）
- 标签质量（Label Quality）

### 7.3 模型解释性

**方法**：
- **SHAP**：SHapley Additive exPlanations
- **LIME**：Local Interpretable Model-agnostic Explanations
- **特征重要性**：特征权重分析

- **迁移学习**：跨设备/跨站点迁移

### 10.3 工程优化

- **自动化 ML**：AutoML（自动模型选择/调优）
- **模型压缩流水线**：自动化模型优化
- **边缘-云协同**：端云协同推理（提升精度）
