# HydroSense Pipeline Overview

## 1. Pipeline 架构概览

HydroSense 数据管道采用**分层解耦**设计，从端侧采集到云端分析，形成完整的数据流转链路。

```
Edge Pipeline → Ingress → Stream Processing → Storage → Analytics → Serving
```

## 2. 端侧 Pipeline（STM32）

### 2.1 数据采集与预处理

**输入**：传感器原始采样值（温度、湿度、雨量、风速等）

**处理步骤**：
1. **去抖/滤波**：硬件噪声抑制
2. **温漂补偿**：环境温度对传感器的影响校正
3. **单位转换**：原始 ADC 值转换为物理单位

**输出**：清洗后的传感器数据

### 2.2 事件触发（Trigger）

**目的**：识别异常事件，触发高频上报

**机制**：
- 阈值触发：超过预设阈值（如雨量 > 阈值）
- 变化率触发：短时间内变化率过大
- 模式匹配：匹配预设异常模式

**输出**：trigger 标志位 + 触发类型

### 2.3 异常抑制（Abnormal Reduce）

**目的**：抑制偶发异常值，避免误报

**算法**：
- 滑动窗口统计（均值/中位数）
- 3-sigma 规则
- 时间序列平滑

**输出**：异常抑制后的数据 + 抑制标志位

### 2.4 缺失填补（Missing Fitting）

**目的**：处理传感器故障/断线导致的数据缺失

**策略**：
- 线性插值（短时缺失）
- 历史均值填充（长时缺失）
- 相邻传感器数据参考（多传感器场景）

**输出**：填补后的数据 + 填补标志位

### 2.5 扰动去除（抗干扰）

**目的**：去除环境干扰（如振动、电磁干扰）

**方法**：
- 频域滤波（FFT + 带通滤波）
- 自适应滤波
- 多传感器融合（交叉验证）

**输出**：去扰动后的数据

### 2.6 小模型推理（CNN Predict）

**目的**：端侧轻量预测（如降雨预测、异常预测）

**模型特点**：
- 轻量 CNN（< 100KB）
- 量化/剪枝优化
- 低功耗推理（< 10mA）

**输出**：预测结果（可选）+ 置信度

### 2.7 打包与缓存

**打包**：
- 组装完整数据包（sensor data + flags + pred + metadata）
- 添加 seq、ts、crc
- 压缩（可选）

**缓存**：
- Ring buffer（内存，最近 N 条）
- Flash（持久化，按时间窗口分段）

**输出**：数据包 + 缓存索引

## 3. 传输 Pipeline

### 3.1 BLE 传输（Mode A）

**流程**：
1. 检查连接状态
2. 选择传输模式（广播 vs GATT）
3. 发送数据包
4. 等待 ACK（关键事件）
5. 失败重试（最多 N 次）
6. 更新 watermark

**失败处理**：
- 写入 ring buffer
- 标记待重传
- 连接恢复后按 seq 补传

### 3.2 离线存储（Mode B）

**流程**：
1. 写入 Flash（分段存储）
2. 更新索引（起止 seq/ts）
3. 定期校验完整性
4. 等待导出触发

**导出流程**：
1. 设备进入维护模式
2. 有线连接
3. 同步 watermark
4. 增量拉取（FETCH_RANGE）
5. 校验完整性
6. 更新 watermark

### 3.3 有线汇聚（Mode C）

**流程**：
1. Base Station 扫描设备
2. 建立连接（串口/USB/RS485）
3. 批量拉取增量数据
4. 站点级聚合
5. 统一上传云端

## 4. 云端 Ingress Pipeline

### 4.1 接入层

**输入**：来自 Gateway/Base Station 的数据包

**处理**：
1. **协议解析**：MQTT/HTTP payload 解析
2. **身份认证**：设备证书/密钥验证
3. **路由**：按 tenant_id/site_id 路由到对应 topic
4. **Schema 校验**：payload 格式校验
5. **时间戳校验**：ts 合理性检查（不能太旧/太新）
6. **签名校验**：可选的数据完整性校验

**输出**：验证通过的数据包 + 路由信息

**失败处理**：
- 认证失败 → 拒绝 + 告警
- Schema 错误 → 记录日志 + 丢弃
- 时间戳异常 → 标记异常 + 可选修复

### 4.2 消息队列

**目的**：解耦接入与处理，缓冲峰值流量

**分区策略**：
- 按 tenant_id 分区（租户隔离）
- 按 site_id 子分区（站点隔离）
- 按 device_id 哈希（负载均衡）

**特性**：
- 至少一次投递（QoS 1）
- 按租户限流
- 死信队列（处理失败的消息）

## 5. 流处理 Pipeline（Streaming QC）

### 5.1 去重处理

**输入**：来自消息队列的数据包

**去重策略**：
- 主键：(device_id, seq)
- 窗口：滑动窗口（最近 N 条）
- 存储：Redis/Bloom Filter

**输出**：去重后的数据包

### 5.2 乱序处理

**问题**：网络延迟导致数据包乱序到达

**处理**：
- 允许小窗口乱序（如 ±5 分钟）
- 按 ts 排序
- 超窗口的数据标记为"延迟数据"

**输出**：按时间排序的数据流

### 5.3 数据质量检查

**检查项**：
- **缺失检测**：seq 不连续 → 标记缺失
- **重复检测**：相同 (device_id, seq) → 去重
- **越界检测**：传感器值超出合理范围
- **单位异常**：单位不匹配
- **延迟统计**：ts 与接收时间的差值

**输出**：质量标记 + 异常记录

### 5.4 实时特征计算

**特征类型**：
- **统计特征**：滑窗均值/方差/最大值/最小值
- **雨强特征**：瞬时雨强、累计雨量
- **频域特征**：FFT 功率谱、主频
- **扰动指标**：信噪比、稳定性指标

**窗口策略**：
- 短窗口（1 分钟）：实时告警
- 中窗口（1 小时）：趋势分析
- 长窗口（1 天）：模式识别

**输出**：特征向量 + 时间窗口信息

### 5.5 告警评估

**告警规则**：
- **离线告警**：设备超过 N 分钟未上报
- **传感器漂移**：长期趋势异常
- **异常雨强**：超过阈值
- **疑似故障**：数据质量持续恶化

**评估流程**：
1. 规则匹配
2. 阈值判断
3. 告警去重（避免重复告警）
4. 告警升级（严重告警）

**输出**：告警事件 + 告警级别

### 5.6 双写输出

**Raw Stream**：
- 原始数据（含所有字段）
- 用途：追溯、审计、回放
- 存储：时序库（原始表）

**Clean/Feature Stream**：
- 清洗后数据 + 特征
- 用途：看板、分析、告警
- 存储：时序库（清洗表 + 特征表）

## 6. 存储 Pipeline

### 6.1 时序数据写入

**写入策略**：
- **批量写入**：按时间窗口批量写入（提高吞吐）
- **分区策略**：按时间分区（如按天/按月）
- **压缩**：列式压缩（如 Parquet）

**存储层级**：
- **热数据**：最近 7 天（原始采样频率）
- **温数据**：7-30 天（分钟级聚合）
- **冷数据**：30 天以上（小时级/天级聚合）

**输出**：写入确认 + 存储位置

### 6.2 元数据更新

**更新内容**：
- 设备最后在线时间
- 设备健康度（基于数据质量）
- 设备版本信息
- 站点统计（设备数、数据量）

**更新策略**：
- 异步更新（不阻塞主流程）
- 批量更新（减少数据库压力）

### 6.3 对象存储写入

**存储内容**：
- 固件包
- 端侧日志
- 离线报表
- 模型文件

**写入流程**：
1. 文件上传
2. 计算 hash
3. 存储到对象存储
4. 更新元数据库（文件路径、hash、大小）

## 7. 分析 Pipeline

### 7.1 实时分析

**查询类型**：
- 实时看板：最近 N 分钟的数据聚合
- 实时告警：流式规则匹配
- 实时统计：在线设备数、数据量

**技术栈**：
- 流式 SQL（如 Flink SQL）
- 时序数据库查询（如 InfluxDB）

### 7.2 离线分析（Batch ETL）

**日报生成**：
- 输入：当日原始数据
- 处理：按站点/设备聚合（均值/最大值/累计值）
- 输出：日报报表（JSON/CSV）

**月报生成**：
- 输入：当月数据
- 处理：趋势分析、异常统计、设备健康度评估
- 输出：月报报表

**校准数据生成**：
- 输入：历史数据 + 校准规则
- 处理：传感器校准参数计算
- 输出：校准参数（下发到设备）

**模型训练数据回放**：
- 输入：历史数据 + 标签
- 处理：特征工程、数据增强
- 输出：训练数据集（用于云端模型训练）

### 7.3 聚合查询服务

**查询类型**：
- **时间范围查询**：按设备/站点查询指定时间范围的数据
- **聚合查询**：按分钟/小时/天聚合
- **下采样查询**：降低采样频率（如 1 秒 → 1 分钟）

**优化策略**：
- 预聚合表（如按小时预聚合）
- 缓存热点查询
- 查询路由（热数据 → 时序库，冷数据 → 对象存储）

## 8. Serving Pipeline

### 8.1 Query API

**接口**：
- `GET /api/v1/devices/{device_id}/telemetry?start={ts}&end={ts}&interval={interval}`
- `GET /api/v1/sites/{site_id}/telemetry?start={ts}&end={ts}&aggregation={agg}`

**处理流程**：
1. 参数校验
2. 权限检查（租户隔离）
3. 查询时序库
4. 数据聚合/下采样
5. 返回 JSON

### 8.2 Dashboard Pipeline

**数据流**：
1. 前端请求实时数据
2. 后端查询时序库/缓存
3. 数据格式化
4. WebSocket 推送（实时更新）

**缓存策略**：
- 最近 1 小时数据缓存（Redis）
- 定时刷新（如每 10 秒）

### 8.3 Alert Pipeline

**告警触发**：
1. 流处理检测到异常
2. 匹配告警规则
3. 生成告警事件
4. 通知订阅者（短信/邮件/企业微信）

**告警去重**：
- 相同告警在 N 分钟内只发送一次
- 告警升级机制（持续异常 → 升级告警级别）

## 9. 可观测性 Pipeline

### 9.1 指标收集

**端侧指标**：
- 采样频率
- 缓存使用率
- 传输成功率
- 电池电量

**云端指标**：
- 接入 QPS
- 处理延迟
- 存储延迟
- 查询 QPS

### 9.2 链路追踪

**追踪点**：
- 端侧采样 → 传输 → 云端接收 → 处理 → 存储 → 查询

**追踪 ID**：贯穿全链路的 trace_id

### 9.3 SLO 监控

**关键 SLO**：
- **数据延迟**：端侧采样到云端入库 < 5 秒（P99）
- **缺失率**：数据缺失率 < 0.1%
- **告警响应时间**：异常事件到告警触发 < 30 秒

**告警**：SLO 不达标时触发运维告警

## 10. 失败处理与容错

### 10.1 端侧容错

- **传输失败**：写入缓存，等待重传
- **处理失败**：降级到规则模式（跳过 CNN）
- **存储失败**：使用备用存储区域

### 10.2 云端容错

- **接入层失败**：消息队列缓冲
- **处理层失败**：死信队列 + 重试
- **存储层失败**：降级到对象存储（raw 数据）
- **查询失败**：返回缓存数据或错误提示

### 10.3 降级策略

- **实时流处理降级**：只写 raw 数据，延迟处理
- **查询降级**：返回预聚合数据（可能不是最新）
- **告警降级**：只发送关键告警
