---
title: "Progressive Resolution Generation"
description: "Speed up FLUX.1 image and Wan T2V video generation up to 2.32× with spectral progressive resolution growing."
---

Progressive resolution growing runs early denoising steps at a coarser latent resolution and spectrally upsamples the latent before the full-resolution steps. This reduces the quadratic attention cost of the DiT transformer, yielding up to **1.63× denoising speedup** on FLUX.1 and up to **2.32× on Wan 2.1 T2V** with no perceptible quality loss.

Based on [Spectral Progressive Diffusion (arXiv 2605.18736)](https://arxiv.org/abs/2605.18736).

## Overview

DiT attention is O(n²) in sequence length. Running the first N denoising steps at half the spatial resolution cuts the attention cost to ~6% for those steps.

The transition point — how many steps to run at each resolution — is computed from the **Bayes-optimal frequency-activation criterion**: frequencies that cannot be resolved at the coarse scale are not denoised there. This makes the speedup lossless by construction.

| Model | Full-res tokens | Half-res tokens | Token-step ratio |
|-------|----------------|----------------|-----------------|
| FLUX.1 1024×1024 | 4,096 | 1,024 | 4.0× |
| Wan 2.1 T2V 480×832 (81 frames) | 6,240 | 1,560 | 4.0× |

## Basic Usage

### FLUX.1 (image)

Add `--progressive-mode dct_rewind` and optionally `--progressive-levels` and `--progressive-delta` to any `sglang generate` command. No other changes are required.

```bash
sglang generate \
    --model-path black-forest-labs/FLUX.1-dev \
    --prompt "A serene mountain lake at golden hour, photorealistic" \
    --num-inference-steps 50 \
    --dit-cpu-offload false \
    --progressive-mode dct_rewind \
    --progressive-levels 1 \
    --progressive-delta 0.05
```

> **Tip:** Add `--dit-cpu-offload false` to keep the transformer GPU-resident. With CPU offload each step pays ~0.4 s for PCIe transfer regardless of sequence length, which dilutes the speedup.

### Wan 2.1 T2V (video)

Progressive generation is supported for `Wan-AI/Wan2.1-T2V-1.3B-Diffusers` and `Wan-AI/Wan2.1-T2V-14B-Diffusers`. Pass `--guidance-scale 5.0 --flow-shift 5.0` to match the reference configuration.

```bash
sglang generate \
    --model-path Wan-AI/Wan2.1-T2V-1.3B-Diffusers \
    --prompt "A cheetah sprinting across the Serengeti at sunset, slow motion, photorealistic" \
    --num-inference-steps 50 \
    --num-frames 81 \
    --height 480 \
    --width 832 \
    --guidance-scale 5.0 \
    --flow-shift 5.0 \
    --dit-cpu-offload false \
    --progressive-mode dct_rewind \
    --progressive-levels 1 \
    --progressive-delta 0.05
```

> **Note:** Progressive generation grows only the **spatial** H×W dimensions. The temporal dimension T (number of latent frames) is kept fixed across all stages — the coarser stage produces the same number of frames at lower spatial resolution.

## Parameters

| Parameter | CLI flag | Default | Description |
|-----------|----------|---------|-------------|
| `progressive_mode` | `--progressive-mode` | `"fullres"` | `"fullres"` disables (identical to standard generation). `"dct_rewind"` enables spectral upsample with scheduler rewind (recommended). `"dct"` enables upsample without rewind. |
| `progressive_levels` | `--progressive-levels` | `1` | Number of resolution halvings. `1` = one coarse stage (64×64 latent → 128×128). `2` = two coarse stages (32×32 → 64×64 → 128×128). |
| `progressive_delta` | `--progressive-delta` | `0.01` | Noise-dominated tolerance δ. Controls how many steps run at the coarse resolution. Higher δ = more coarse steps = faster but less conservative. |

### Choosing delta

**FLUX.1** (1024×1024, 50 steps):

| δ | Coarse steps (50 total) | Denoising speedup |
|---|------------------------|-------------------|
| `0.01` | 18 @ 64² + 32 @ 128² | **1.32×** |
| `0.05` | 28 @ 64² + 22 @ 128² | **1.63×** |

**Wan 2.1 T2V** (480×832, 81 frames, 50 steps, flow\_shift=5.0):

| δ | Coarse steps (50 total) | Denoising speedup |
|---|------------------------|-------------------|
| `0.01` | 23 @ 30×52 + 27 @ 60×104 | **1.65×** |
| `0.02` | 27 @ 30×52 + 23 @ 60×104 | **1.86×** |
| `0.05` | 33 @ 30×52 + 17 @ 60×104 | **2.32×** |
| `0.10` | 37 @ 30×52 + 13 @ 60×104 | **2.78×** |

For most prompts `0.05` is recommended — it gives a large speedup with no visible quality degradation. `0.10` provides the maximum speedup but should be validated on motion-heavy scenes.

## Python API

### FLUX.1

```python
from sglang.multimodal_gen import DiffGenerator

gen = DiffGenerator.from_pretrained(
    model_path="black-forest-labs/FLUX.1-dev",
    dit_cpu_offload=False,
)

result = gen.generate(sampling_params_kwargs={
    "prompt": "A serene mountain lake at golden hour, photorealistic",
    "num_inference_steps": 50,
    "height": 1024,
    "width": 1024,
    "progressive_mode": "dct_rewind",
    "progressive_levels": 1,
    "progressive_delta": 0.05,
})
```

## Benchmarks

Hardware: RTX A6000 48 GB, `--dit-cpu-offload false`. Timing = denoising loop only.

### FLUX.1 (1024×1024, 50 steps)

| Config | Stage split | Denoise | Speedup |
|--------|-------------|---------|---------|
| Fullres (baseline) | 50 @ 128² | 36.65 s | 1.00× |
| dct_rewind L1 δ=0.01 | 18@64² + 32@128² | 27.67 s | **1.32×** |
| dct_rewind L1 δ=0.05 | 28@64² + 22@128² | 22.58 s | **1.62×** |
| dct_rewind L2 δ=0.01 | 10@32² + 8@64² + 32@128² | 26.48 s | **1.38×** |

### Wan 2.1 T2V 1.3B (480×832, 81 frames, 50 steps, guidance=5.0, flow\_shift=5.0)

| Config | Stage split | Denoise | Speedup |
|--------|-------------|---------|---------|
| Fullres (baseline) | 50 @ 60×104 | 266.8 s | 1.00× |
| dct_rewind L1 δ=0.01 | 23@30×52 + 27@60×104 | 161.5 s | **1.65×** |
| dct_rewind L1 δ=0.02 | 27@30×52 + 23@60×104 | 142.7 s | **1.86×** |
| dct_rewind L1 δ=0.05 | 33@30×52 + 17@60×104 | 114.7 s | **2.32×** |
| dct_rewind L1 δ=0.10 | 37@30×52 + 13@60×104 | 95.9 s | **2.78×** |

Wan speedups exceed the token-step prediction because the half-resolution attention (1,560 tokens) is quadratically cheaper than full-resolution (6,240 tokens), giving super-linear gains per token saved.

## Limitations

- **Supported models.** Progressive generation is currently implemented for FLUX.1, FLUX.2, and Wan 2.1 T2V pipelines.
- **Sequence parallelism incompatible.** Cannot be combined with `--ulysses-degree` or `--ring-degree`. The stage raises a `RuntimeError` if SP is enabled.
- **torch.compile incompatible.** Compiled kernels have a fixed sequence length; the resolution transition causes a recompile or error. Use progressive without `--enable-torch-compile`.
- **Cache-DiT incompatible.** Cache-DiT indexes its step cache by step count; the stage transition resets the index.
- **Wan T2V: spatial alignment.** Wan's patch embedding requires even spatial latent dimensions at every stage. The pipeline automatically snaps `height` and `width` down to the nearest valid multiple when needed (e.g., 720→704 for L=1). The output video reflects the adjusted resolution.
