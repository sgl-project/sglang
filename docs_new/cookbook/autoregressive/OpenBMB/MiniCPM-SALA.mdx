## 1. Model introduction

[MiniCPM-SALA](https://huggingface.co/openbmb/MiniCPM-SALA) is an OpenBMB long-context language model that combines Sparse Attention and Linear Attention. The model is designed for efficient long-context inference and million-token context modeling.

**Key features:**

- **Hybrid attention**: combines sparse attention layers with Lightning Attention layers.
- **Long-context inference**: targets efficient inference at very long context lengths.
- **HyPE positional encoding**: improves short-context and long-context behavior in the same model.
- **SGLang support**: the Hugging Face model card includes SGLang installation and launch instructions for the MiniCPM-SALA runtime path.

**License:** [Apache 2.0](https://www.apache.org/licenses/LICENSE-2.0).

## 2. Requirements

MiniCPM-SALA uses custom model code and CUDA extensions. The Hugging Face model card lists these requirements for the SGLang path:

- CUDA 12.x or higher
- `gcc` / `g++`
- `uv`
- Python 3.12 for the provided environment setup

## 3. SGLang installation

The current MiniCPM-SALA instructions use OpenBMB's SGLang branch and setup script:

```bash Command
git clone -b minicpm_sala https://github.com/OpenBMB/sglang.git
cd sglang
bash install_minicpm_sala.sh
```

To use a PyPI mirror, pass the mirror URL to the script:

```bash Command
bash install_minicpm_sala.sh https://mirrors.tuna.tsinghua.edu.cn/pypi/web/simple
```

The setup script creates a Python 3.12 virtual environment, installs SGLang, and compiles the MiniCPM-SALA CUDA dependencies.

## 4. Model deployment

Activate the environment created by the setup script:

```bash Command
source sglang_minicpm_sala_env/bin/activate
```

Set the model path and launch the server:

```bash Command
MODEL_PATH=/path/to/your/MiniCPM-SALA

python3 -m sglang.launch_server \
  --model ${MODEL_PATH} \
  --trust-remote-code \
  --disable-radix-cache \
  --attention-backend minicpm_flashinfer \
  --chunked-prefill-size 8192 \
  --max-running-requests 32 \
  --skip-server-warmup \
  --port 31111 \
  --dense-as-sparse
```

### 4.1 Launch arguments

| Argument | Purpose |
| --- | --- |
| `--trust-remote-code` | Allows custom model code from the model repository. |
| `--disable-radix-cache` | Disables RadixAttention prefix caching for this runtime path. |
| `--attention-backend minicpm_flashinfer` | Uses the MiniCPM FlashInfer backend. |
| `--chunked-prefill-size 8192` | Sets the chunked prefill size. |
| `--max-running-requests 32` | Sets the maximum number of concurrent running requests. |
| `--skip-server-warmup` | Skips server warmup. |
| `--dense-as-sparse` | Uses dense-as-sparse mode. |

## 5. Model invocation

Send an OpenAI-compatible chat completion request:

```bash Command
curl http://localhost:31111/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "/path/to/your/MiniCPM-SALA",
    "messages": [
      {
        "role": "user",
        "content": "What is the capital of France?"
      }
    ],
    "temperature": 0.9
  }'
```

The Hugging Face model card recommends `temperature=0.9` for inference.

## 6. Troubleshooting

If CUDA extension compilation fails:

- Check that CUDA 12.x or higher is installed with `nvcc --version`.
- Check that `gcc` and `g++` are available.
- If `CXX` points to `clang++ -pthread`, set it to `g++` before compiling.
