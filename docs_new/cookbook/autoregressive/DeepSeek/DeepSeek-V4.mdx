## 1. Model Introduction

**DeepSeek-V4** is the next-generation Mixture-of-Experts model from DeepSeek, released 2026-04-24 under an **MIT License**. It ships as two Instruct repos (one per variant) plus matching Base repos:

| Variant                                                                          | Total params | Active (MoE) | Use                                                  |
| -------------------------------------------------------------------------------- | -----------: | -----------: | ---------------------------------------------------- |
| **[DeepSeek-V4-Flash](https://huggingface.co/deepseek-ai/DeepSeek-V4-Flash)**    | **284B**     | 13B          | single-node serving: B200 / GB300 / H200 on 4 GPUs   |
| **[DeepSeek-V4-Pro](https://huggingface.co/deepseek-ai/DeepSeek-V4-Pro)**        | **1.6T**     | 49B          | high-capacity: B200 8 GPU / GB300 4 GPU / H200 16 GPU (2 nodes)      |

The Instruct repos ship **FP4 MoE experts + FP8 attention / dense** (one mixed-precision checkpoint covers all GPUs that support FP4). The Base (pre-trained only) variants — `DeepSeek-V4-Flash-Base`, `DeepSeek-V4-Pro-Base` — ship pure FP8 mixed and are **not** for chat / tool calling.

**Key Features** (per the official model card):

- **Hybrid Attention Architecture** — combines Compressed Sparse Attention (CSA) and Heavily Compressed Attention (HCA) for long-context efficiency. At 1M-token context, DeepSeek-V4-Pro uses only ~27% of per-token inference FLOPs and ~10% of KV cache compared with DeepSeek-V3.2.
- **Manifold-Constrained Hyper-Connections (mHC)** — strengthens residual connections, improving signal-propagation stability across layers while preserving expressivity.
- **Muon optimizer** — faster convergence and greater training stability.
- **Context length: 1M tokens**; pre-trained on 32T+ diverse, high-quality tokens.
- **Three reasoning modes**: *Non-think* (fast, intuitive responses), *Think High* (conscious logical analysis, slower but more accurate), *Think Max* (push reasoning to its fullest extent). Recommend a ≥ 384K context window when running Think Max.
- Ships with a dedicated `encoding_dsv4.encode_messages` Python encoder + DSML tool-call grammar (`<｜DSML｜tool_calls>` / `<｜DSML｜invoke>` / `<｜DSML｜parameter>`).

**Recommended Generation Parameters:** `temperature=1.0`, `top_p=1.0` (per the official model card).

**License:** MIT.

**Resources:**

- HuggingFace: [DeepSeek-V4-Flash](https://huggingface.co/deepseek-ai/DeepSeek-V4-Flash), [DeepSeek-V4-Pro](https://huggingface.co/deepseek-ai/DeepSeek-V4-Pro)

**TP / DP sizing**

| Model Size | B200 | GB300 | H200                |
| ---------- | ---- | ----- | ------------------- |
| Flash      | TP=4 | TP=4  | TP=4                |
| Pro (1.6T) | TP=8 | TP=4  | TP=16 (2-node)      |

- `--dp` commonly matches `--tp` when `--enable-dp-attention` is set.
- PD-disagg TP sizing (per role) is smaller than non-PD: B200 Flash PD = TP=2 per role; otherwise same as non-PD.
- H200 Pro is multi-node: run the same command on each of the 2 nodes with `<node-rank>` set to `0` on the head node, `1` on the second; `<node0-ip>` is the head node's IP (reachable from both). The generator emits these as placeholders.

**Concurrency & DeepEP dispatch buffer**

Three knobs are coupled and must satisfy one invariant:

- `SGLANG_DEEPEP_NUM_MAX_DISPATCH_TOKENS_PER_RANK` — DeepEP per-rank dispatch-buffer capacity (tokens). Too small → `deep_ep.cpp:1105` assertion fires. Too big → wasted NVSHMEM symm-alloc VRAM.
- `--cuda-graph-max-bs` — largest batch size CUDA graphs get captured for. Caps the CG replay pool; batches above this fall back to non-CG.
- `--max-running-requests` — scheduler ceiling on concurrent in-flight requests.

**Invariant (must hold):**

```
max-running-requests × MTP_draft_tokens  ≤  SGLANG_DEEPEP_NUM_MAX_DISPATCH_TOKENS_PER_RANK
```

where `MTP_draft_tokens` = `--speculative-num-draft-tokens` (4 for low-latency, 2 for balanced, 1 for max-throughput / recipes without MTP). Violating the invariant blows the DeepEP dispatch buffer at steady-state load.

Typical configurations (all picked by the generator automatically):

| Recipe           | MTP draft | env (dispatch cap) | cg-max-bs | max-running-reqs |
| ---------------- | --------: | -----------------: | --------: | ---------------: |
| low-latency (H200 Pro)   | 4 | 128  | 32  | 64   |
| balanced (H200)  |         2 | 256                | 128       | 128              |
| balanced (Blackwell Flash) | 2 | 1024            | sglang default | sglang default |
| max-throughput (H200)    | 1 | 256  | 128 | 256  |
| max-throughput (Blackwell Flash) | 1 | 1024    | sglang default | sglang default |

If you tune these yourself, move all three together — changing one without the others is how you hit either the `deep_ep.cpp:1105` crash (too-small env) or OOM during CG capture (too-large env).

**MTP (Multi-Token Prediction, EAGLE)**

- `low-latency`: steps=3, draft-tokens=4 → largest win at bs=1.
- `balanced`: steps=1, draft-tokens=2 → gentler MTP, reduces throughput hit at higher batch.
- `max-throughput`: MTP disabled — at saturation the verify step costs more than it saves.
- MTP currently requires `SGLANG_ENABLE_SPEC_V2=1`.

**DeepEP**

- All `balanced` / `max-throughput` / `cp` recipes use `--moe-a2a-backend deepep` on every platform. For `pd-disagg`, only Blackwell (B200 / GB300) uses DeepEP; H200 PD doesn't. For `low-latency`, DeepEP is only used by the H200-Pro multi-node exception; every other low-latency cell skips it.
- On single-node, non-TBO recipes it is beneficial to raise DeepEP `num_sms` from the default 20 to 96 (verified on B200 FP4 prefill: chunk 16k → +19% tok/s, chunk 8k → +23% tok/s). Do NOT combine with `--enable-two-batch-overlap`.
- Set `SGLANG_DEEPEP_NUM_MAX_DISPATCH_TOKENS_PER_RANK` to match peak CUDA-graph load: `cuda-graph-max-bs × MTP_draft_tokens`. Too small trips `deep_ep.cpp:1105`; too large wastes VRAM on NVSHMEM symm buffers.

**Context Parallel (long prompts)**

- Requires `--enable-nsa-prefill-context-parallel --nsa-prefill-cp-mode round-robin-split`, larger `--chunked-prefill-size` (16384), lower `--mem-fraction-static` (0.78).
- Currently does **not** combine with MTP in the validated recipes.

**GB300 specifics**

- GB300 image is not yet on Docker Hub (being pushed). Once published it is expected to be a separate tag from B200's `deepseek-v4-blackwell` due to the CUDA-13 / Grace-Blackwell userland differences.
- For PD-disagg over MNNVL (mooncake-over-NVLink): export `SGLANG_MOONCAKE_CUSTOM_MEM_POOL=True` (see [PD Disaggregation docs](../../../docs/advanced_features/pd_disaggregation) for supported values) and drop the `--disaggregation-ib-device` flag. Additional NCCL / transport knobs may be required depending on your cluster's NVLink topology.
- DRA `compute-domain-channel` claim is required; without it, DeepEP CUDA-graph capture fails with `operation not permitted`.

<a id="hopper-note" />

**Hopper (H200) note — open item**

The public `deepseek-ai/DeepSeek-V4-Flash` / `-Pro` repos ship with **MoE experts in FP4** (plus FP8 for the rest). Hopper GPUs don't have FP4 MoE kernels, so the public path on H200 depends on either (a) SGLang auto-dequantizing FP4 experts to FP8 on load, or (b) a separate HF upload of an FP8-only Instruct checkpoint. The H200 Docker image is currently being pushed but not yet public. Internally we use a converted FP8-only checkpoint (`DeepSeek-V4-HF-FP8-cvt` in the sunrise-nda allinone paths). Until the H200 tag is live, the H200 recipes in this cookbook are **pre-release reference** — commands are verified against the internal source of truth, but the public user journey on H200 is not yet complete.

### 4.1 Basic Usage

For basic API usage and request examples, see:

Once the server is running (for example via the command generator above), send a request:

```shell Command
curl http://localhost:30000/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "deepseek-ai/DeepSeek-V4-Flash",
    "messages": [{"role": "user", "content": "What is 15% of 240?"}]
  }'
```

> **PD-Disagg note**: if you deployed with the `pd-disagg` recipe from the generator above, the prefill server is on port `30000`, the decode server on `30001`, and the **router** on port `8000` — client traffic should target `http://localhost:8000`, not `:30000`.

### 4.2 Advanced Usage

#### 4.2.1 Reasoning Parser

Enable the `deepseek-v4` reasoning parser to separate thinking from the final answer into `reasoning_content` vs `content`.

```shell Command
sglang serve \
  --model-path deepseek-ai/DeepSeek-V4-Flash \
  --reasoning-parser deepseek-v4 \
  --tp 4 \
  --host 0.0.0.0 \
  --port 30000
```

**Streaming with Thinking Process:**

```python Example
from openai import OpenAI

client = OpenAI(
    base_url="http://localhost:30000/v1",
    api_key="EMPTY"
)

response = client.chat.completions.create(
    model="deepseek-ai/DeepSeek-V4-Flash",
    messages=[
        {"role": "user", "content": "Solve this problem step by step: What is 15% of 240?"}
    ],
    max_tokens=2048,
    extra_body={"chat_template_kwargs": {"thinking": True}},
    stream=True,
)

thinking_started = False
has_thinking = False
has_answer = False

for chunk in response:
    if not chunk.choices:
        continue
    delta = chunk.choices[0].delta

    if getattr(delta, "reasoning_content", None):
        if not thinking_started:
            print("=============== Thinking =================", flush=True)
            thinking_started = True
        has_thinking = True
        print(delta.reasoning_content, end="", flush=True)

    if delta.content:
        if has_thinking and not has_answer:
            print("\n=============== Content =================", flush=True)
            has_answer = True
        print(delta.content, end="", flush=True)

print()
```

#### 4.2.2 Tool Calling

Enable the `deepseekv4` tool-call parser to surface structured tool calls via `message.tool_calls`.

**Deployment Command:**

```shell Command
sglang serve \
  --model-path deepseek-ai/DeepSeek-V4-Flash \
  --tool-call-parser deepseekv4 \
  --reasoning-parser deepseek-v4 \
  --tp 4 \
  --host 0.0.0.0 \
  --port 30000
```

**Python Example (with Thinking Process):**

```python Example
from openai import OpenAI

client = OpenAI(
    base_url="http://localhost:30000/v1",
    api_key="EMPTY"
)

tools = [
    {
        "type": "function",
        "function": {
            "name": "get_weather",
            "description": "Get the current weather for a location",
            "parameters": {
                "type": "object",
                "properties": {
                    "location": {"type": "string", "description": "The city name"},
                    "unit": {"type": "string", "enum": ["celsius", "fahrenheit"]},
                },
                "required": ["location"],
            },
        },
    }
]

response = client.chat.completions.create(
    model="deepseek-ai/DeepSeek-V4-Flash",
    messages=[{"role": "user", "content": "What's the weather in Beijing?"}],
    tools=tools,
    extra_body={"chat_template_kwargs": {"thinking": True}},
    stream=True,
)

thinking_started = False
has_thinking = False
tool_calls_accumulator = {}

for chunk in response:
    if not chunk.choices:
        continue
    delta = chunk.choices[0].delta

    if getattr(delta, "reasoning_content", None):
        if not thinking_started:
            print("=============== Thinking =================", flush=True)
            thinking_started = True
        has_thinking = True
        print(delta.reasoning_content, end="", flush=True)

    if getattr(delta, "tool_calls", None):
        if has_thinking and thinking_started:
            print("\n=============== Content =================\n", flush=True)
            thinking_started = False
        for tool_call in delta.tool_calls:
            index = tool_call.index
            if index not in tool_calls_accumulator:
                tool_calls_accumulator[index] = {"name": None, "arguments": ""}
            if tool_call.function:
                if tool_call.function.name:
                    tool_calls_accumulator[index]["name"] = tool_call.function.name
                if tool_call.function.arguments:
                    tool_calls_accumulator[index]["arguments"] += tool_call.function.arguments

    if delta.content:
        print(delta.content, end="", flush=True)

for index, tool_call in sorted(tool_calls_accumulator.items()):
    print(f"Tool Call: {tool_call['name']}")
    print(f"   Arguments: {tool_call['arguments']}")

print()
```
