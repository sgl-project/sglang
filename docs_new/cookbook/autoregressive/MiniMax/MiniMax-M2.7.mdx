### Configuration notes

- **Trust remote code** is required for MiniMax model loading (`--trust-remote-code`), and `--mem-fraction-static 0.85` is the recommended KV-cache static fraction on GPU recipes.
- **Strategies are GPU-budget tiers**: Low-Latency = 2 GPUs (TP=2, available on AMD and GB300), Balanced = 4 GPUs (TP=4), High-Throughput = 8 GPUs (TP=8 + EP=8). GB300 (275 GB/die) can host the model at TP=2 and tops out at single-node TP=4; the 2-GPU tier is hidden on the other NVIDIA cards, which start at TP=4.
- **NVFP4** (`nvidia/MiniMax-M2.7-NVFP4`) requires Blackwell — it is offered only on B200 and GB300.
- **NVIDIA H200/B200/GB300** add fused QK-norm (`SGLANG_USE_FUSED_PARALLEL_QKNORM=1`) and `--enable-flashinfer-allreduce-fusion`. Blackwell (B200/GB300) additionally pins `--moe-runner-backend flashinfer_trtllm_routed`; the FP8 path adds `--fp8-gemm-backend flashinfer_trtllm --dtype bfloat16`.
- **AMD (MI300X/MI325X/MI355X)** scale EP with TP (EP=TP) and serve with `--kv-cache-dtype fp8_e4m3` (for KV-cache memory efficiency) and `--attention-backend triton`.
- **Intel Xeon CPU**: run on a Granite Rapids (GNR) AP 2-socket server with `--device cpu --disable-overlap-schedule --tp 6`. See the `Notes` in the [SGLang CPU server document](../../../docs/hardware-platforms/cpu_server#launch-of-the-serving-engine) for TP and NUMA-binding guidance.
- **Thinking and tool calling** are exposed in the [Playground](#playground) (the `--reasoning-parser minimax-append-think` and `--tool-call-parser minimax-m2` flags), not baked into the Deploy command — toggle them there.

### Basic chat completion

<Accordion title="Simple completion (Python)">
```python Example
from openai import OpenAI

client = OpenAI(
    base_url="http://localhost:30000/v1",
    api_key="EMPTY"
)

response = client.chat.completions.create(
    model="MiniMaxAI/MiniMax-M2.7",
    messages=[
        {"role": "system", "content": "You are a helpful assistant."},
        {"role": "user", "content": "Who won the world series in 2020?"}
    ],
    max_tokens=1024
)

print(response.choices[0].message.content)
```

### Reasoning parser (thinking mode)

MiniMax-M2.7 supports Thinking mode. With `--reasoning-parser minimax-append-think`, the thinking content is wrapped in `<think>...</think>` tags **inside** the `content` field — parse these tags client-side to separate thinking from the answer. (Enable the Reasoning Parser in the [Playground](#playground) to add the flag to your launch command.)

<Accordion title="Streaming with thinking process (Python)">
```python Example
from openai import OpenAI

client = OpenAI(
    base_url="http://localhost:30000/v1",
    api_key="EMPTY"
)

# Enable streaming to see the thinking process in real-time
response = client.chat.completions.create(
    model="MiniMaxAI/MiniMax-M2.7",
    messages=[
        {"role": "user", "content": "Solve this problem step by step: What is 15% of 240?"}
    ],
    max_tokens=2048,
    stream=True
)

# Process the stream, separating <think>...</think> from content
in_think = False
think_printed_header = False
content_printed_header = False
buffer = ""

for chunk in response:
    if chunk.choices and len(chunk.choices) > 0:
        delta = chunk.choices[0].delta
        if delta.content:
            buffer += delta.content

            while buffer:
                if in_think:
                    # Look for closing </think> tag
                    end_idx = buffer.find("</think>")
                    if end_idx != -1:
                        print(buffer[:end_idx], end="", flush=True)
                        buffer = buffer[end_idx + len("</think>"):]
                        in_think = False
                    else:
                        # Still in thinking, print what we have
                        print(buffer, end="", flush=True)
                        buffer = ""
                else:
                    # Look for opening <think> tag
                    start_idx = buffer.find("<think>")
                    if start_idx != -1:
                        # Print any content before <think>
                        before = buffer[:start_idx]
                        if before:
                            if not content_printed_header:
                                print("=============== Content =================", flush=True)
                                content_printed_header = True
                            print(before, end="", flush=True)
                        buffer = buffer[start_idx + len("<think>"):]
                        in_think = True
                        if not think_printed_header:
                            print("=============== Thinking =================", flush=True)
                            think_printed_header = True
                    else:
                        # No <think> tag, print as content
                        if not content_printed_header and think_printed_header:
                            print("\n=============== Content =================", flush=True)
                            content_printed_header = True
                        print(buffer, end="", flush=True)
                        buffer = ""

print()
```
</Accordion>

### Tool calling

MiniMax-M2.7 supports tool calling. Enable the Tool Call Parser in the [Playground](#playground) to add `--tool-call-parser minimax-m2` to your launch command.

<Accordion title="Tool call request (Python)">
```python Example
from openai import OpenAI

client = OpenAI(
    base_url="http://localhost:30000/v1",
    api_key="EMPTY"
)

# Define available tools
tools = [
    {
        "type": "function",
        "function": {
            "name": "get_weather",
            "description": "Get the current weather for a location",
            "parameters": {
                "type": "object",
                "properties": {
                    "location": {
                        "type": "string",
                        "description": "The city name"
                    },
                    "unit": {
                        "type": "string",
                        "enum": ["celsius", "fahrenheit"],
                        "description": "Temperature unit"
                    }
                },
                "required": ["location"]
            }
        }
    }
]

# Non-streaming request
response = client.chat.completions.create(
    model="MiniMaxAI/MiniMax-M2.7",
    messages=[
        {"role": "user", "content": "What's the weather in Beijing?"}
    ],
    tools=tools
)

message = response.choices[0].message

# Check for tool calls
if message.tool_calls:
    for tool_call in message.tool_calls:
        print(f"Tool Call: {tool_call.function.name}")
        print(f"   Arguments: {tool_call.function.arguments}")
else:
    print(message.content)
```
</Accordion>

<Accordion title="Example Output">
```text Output
Tool Call: get_weather
   Arguments: {"location": "Beijing"}
```
</Accordion>

<Accordion title="Handling tool results (Python)">
```python Example
# After getting the tool call, execute the function
def get_weather(location, unit="celsius"):
    # Your actual weather API call here
    return f"The weather in {location} is 22°{unit[0].upper()} and sunny."

# Send tool result back to the model
messages = [
    {"role": "user", "content": "What's the weather in Beijing?"},
    {
        "role": "assistant",
        "content": None,
        "tool_calls": [{
            "id": "call_123",
            "type": "function",
            "function": {
                "name": "get_weather",
                "arguments": '{"location": "Beijing", "unit": "celsius"}'
            }
        }]
    },
    {
        "role": "tool",
        "tool_call_id": "call_123",
        "content": get_weather("Beijing", "celsius")
    }
]

final_response = client.chat.completions.create(
    model="MiniMaxAI/MiniMax-M2.7",
    messages=messages
)

print(final_response.choices[0].message.content)
```
</Accordion>

<Accordion title="Example Output">
```text Output
The weather in Beijing is currently 22°C and sunny.
```
