## 1. Model Introduction

**Command A+** (`command-a-plus-05-2026`) is Cohere's open-weights flagship: a **218B-parameter sparse Mixture-of-Experts** model that activates just **25B parameters per token**, built for agentic, multilingual, and reasoning-heavy enterprise workloads. It is a **vision-language model** (`image-text-to-text`) — a SigLIP vision encoder feeds a `cohere2_moe` text backbone of 32 layers with **128 experts (8 routed + 4 shared per token)**, grouped-query attention (128 query / 8 KV heads), and an interleaved **sliding-window (4096) + global attention** pattern in a 3:1 ratio. It understands **48 languages**, supports **tool / function calling** and citation-grounded answers, and has an **extended-thinking** mode. Context length is up to **200K tokens** (128K input / 64K output recommended). Released May 2026 under the **Apache 2.0** license.

**Multimodal serving (image + text).** When you serve images at concurrency, enable CUDA-IPC transport for the vision features and raise the feature-cache budget — the default 1024 MiB exhausts under sustained multimodal load and silently falls back to the slower path:

```bash Command
SGLANG_USE_CUDA_IPC_TRANSPORT=1 SGLANG_MM_FEATURE_CACHE_MB=8192 \
sglang serve --model-path CohereLabs/command-a-plus-05-2026-w4a4 \
  --tp 1 --trust-remote-code --moe-runner-backend flashinfer_trtllm \
  --host 0.0.0.0 --port 30000
```

**Context length.** The benchmarks below were measured at a bounded `--context-length 12288`. The model's native maximum is 200K; omit `--context-length` to let SGLang size the KV pool from available memory. Because most layers use a 4096-token sliding window (only every 4th layer is full-attention), long-context KV stays modest, so large contexts are practical on a single node.

**Memory fraction.** Recipes were verified at the default memory fraction; raising `--mem-fraction-static` to `0.9` changed throughput by `<0.5%` at the tested concurrencies (these workloads are MoE-compute-bound, not KV-bound).

**Hardware footprint — two recipe families.** **Blackwell (B200 / B300):** BF16 `cutlass` TP=4, FP8 Triton TP=2, W4A4 `flashinfer_trtllm` TP=1 — **B300 is benchmark-verified**; B200 runs the same commands but is **not yet benchmarked** (shown *Not Verified*). **Hopper (H100 / H200):** plain tensor parallelism on the default Triton FusedMoE runner, sized from Cohere's official guidance — **BF16 TP=8, FP8 TP=4**. **H200 is benchmark-verified** (on an 8× H200 node); H100 uses the same recipe but is not yet benchmarked. **W4A4 (NVFP4) is Blackwell-only** — the Hopper dequant path isn't validated, so the Deploy panel greys out W4A4 on H100/H200.

**Current-build note (prefill CUDA graph).** On recent SGLang `main` (verified at commit `20b2817`), Command A+'s `cohere2_moe` crashes during `tc_piecewise` prefill-graph capture (`AssertionError: Index not registered in index_to_user_object_weakref`). The **Hopper cells therefore launch with `--cuda-graph-backend-prefill disabled`** — the decode CUDA graph stays on (throughput is representative), and only the prefill graph is dropped (which raises TTFT on long inputs). The Blackwell numbers were measured on an earlier commit (`7cead0f`) before this regression landed; if you hit the same crash on Blackwell, add the same flag.

**Hierarchical KV cache.** For a 218B model, host-tier KV offload can extend effective cache capacity — toggle it in the **Hierarchical KV Cache** card of the [Playground above](#playground).

### 3.1 Reasoning

Command A+ has an extended-thinking mode. Enable the `cohere_command4` reasoning parser (toggle **Reasoning Parser** in the **Parsers** card of the [Playground above](#playground)) to separate the thinking trace from the final answer. This is a **separate-field** parser: the thinking lands in `message.reasoning_content` and the answer in `message.content`. Thinking is on by default; pass `chat_template_kwargs={"reasoning": False}` to turn it off.

<Accordion title="Reasoning Example (Python)">

```python Example
from openai import OpenAI

client = OpenAI(base_url="http://localhost:30000/v1", api_key="EMPTY")
resp = client.chat.completions.create(
    model="CohereLabs/command-a-plus-05-2026-bf16",
    messages=[{"role": "user", "content": "What is 15% of 240? Think step by step."}],
)
msg = resp.choices[0].message
print("Reasoning:", getattr(msg, "reasoning_content", None))
print("Answer:", msg.content)
```

</Accordion>

### 3.2 Tool Calling

Enable the `cohere_command4` tool-call parser (toggle **Tool Call Parser** in the **Parsers** card of the [Playground above](#playground)) to surface structured tool calls via `message.tool_calls`. Because thinking is on by default, the assistant turn may also populate `message.reasoning_content` — print both so the output isn't a misleading `None`.

<Accordion title="Tool Calling Example (Python)">

```python Example
from openai import OpenAI

client = OpenAI(base_url="http://localhost:30000/v1", api_key="EMPTY")
tools = [{
    "type": "function",
    "function": {
        "name": "get_weather",
        "description": "Get the current weather for a city.",
        "parameters": {
            "type": "object",
            "properties": {"city": {"type": "string", "description": "City name"}},
            "required": ["city"],
        },
    },
}]
resp = client.chat.completions.create(
    model="CohereLabs/command-a-plus-05-2026-bf16",
    messages=[{"role": "user", "content": "What's the weather in Toronto right now?"}],
    tools=tools,
)
msg = resp.choices[0].message
print("Reasoning:", getattr(msg, "reasoning_content", None))
print("Tool calls:", msg.tool_calls)
```

</Accordion>

### 3.3 Vision (Image + Text)

Command A+ accepts image input. Send an image via the OpenAI `image_url` content part (a public URL or a `data:` base64 URI). See the multimodal serving tip in §2 for the CUDA-IPC flags that keep image prefill fast under load.

<Accordion title="Vision Example (Python)">

```python Example
from openai import OpenAI

client = OpenAI(base_url="http://localhost:30000/v1", api_key="EMPTY")
resp = client.chat.completions.create(
    model="CohereLabs/command-a-plus-05-2026-bf16",
    messages=[{
        "role": "user",
        "content": [
            {"type": "text", "text": "Describe this image in one sentence."},
            {"type": "image_url", "image_url": {"url": "https://raw.githubusercontent.com/sgl-project/sglang/main/examples/assets/example_image.png"}},
        ],
    }],
)
print(resp.choices[0].message.content)
```

</Accordion>

<Accordion title="Example Output">

```text Output
A man in a yellow shirt is ironing a shirt on an ironing board mounted on the back of a yellow taxi on a busy city street.
```
