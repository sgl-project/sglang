## 1. Model Introduction

[`NVIDIA Nemotron3-Ultra`](https://huggingface.co/nvidia/NVIDIA-Nemotron-3-Ultra-550B-A55B-BF16) is an open frontier reasoning model in the Nemotron 3 family, built for long-running autonomous agents. It is a **550B-parameter hybrid Transformer-Mamba Mixture-of-Experts** model that activates only **55B parameters** per forward pass, and supports a **1M-token context window** so agents can keep conversation history, tool outputs, and plan state in view across persistent workflows. It is optimized for complex orchestration across coding, deep research, enterprise automation, and EDA use cases.

Architecture and key features:

- **Hybrid Transformer-Mamba architecture (MoE):** combines Mixture-of-Experts with a hybrid Transformer-Mamba stack, enabling efficient routing and sequence modeling in a single model.
- **Long-horizon agentic reasoning:** tuned for agents that plan, call tools, inspect results, recover from failures, and continue working across long task horizons.
- **1M-token context window:** sustains coherent agent state across extended workflows without re-ingestion.
- **BF16 and NVFP4 quantization:** deployable from a multi-node H100 cluster down to a single Blackwell node with NVFP4.
- **Open weights, open data, open recipes:** customizable for domain-specific agents and deployable across your own infrastructure.

**Modalities:** Input: text — Output: text

## 2. Configuration Tips

- **Mamba scheduler strategy:** always launch with `--mamba-scheduler-strategy extra_buffer` (baked into every recipe above). This hybrid Transformer-Mamba model requires the `extra_buffer` strategy for correct scheduling of its Mamba state.

- **Attention backend:** on **H100/H200** SGLang uses the FlashAttention-3 backend by default — no flag needed. On **B200/GB200/B300/GB300** the recipes append `--attention-backend trtllm_mha` (also baked above): the FlashInfer default breaks the overlap scheduler on Blackwell, so `trtllm_mha` is required there.

- **Tensor parallelism:** BF16 runs `--tp 16` on H100/H200 (a 2-node setup — select **Multi-Nodes** in the panel) and `--tp 8` on B200/B300; NVFP4 runs `--tp 4` on all four Blackwell platforms, with `--tp 8` additionally validated on B200/B300 (raise the **TP** knob in the [Playground](#playground)).

- **Multi-node BF16 on H100/H200:** the 16-GPU BF16 setup spans two nodes. Selecting **Multi-Nodes** injects `--dist-init-addr <head-node-ip>:20000 --nnodes 2 --node-rank <0|1>` — run the emitted command on each node with the matching `--node-rank`.

- **DP attention:** by default the attention layers are tensor-parallel (sharded across all TP ranks). Enabling DP attention (the **High-Throughput** strategy, or `--dp <N> --enable-dp-attention`) instead runs attention as `N` data-parallel groups, each serving its own slice of requests with its own KV cache. `--dp` must divide `--tp`; DP=2 is validated for BF16 and DP∈{2,4,8} for NVFP4 (capped by TP). Tune the degree with the **DP-Attention** knob in the Playground.

- **Expert parallel (EP):** this MoE supports only `ep_size == 1` (off) or `ep_size == tp_size`. To enable expert parallelism, set `--ep` equal to `--tp` (the **High-Throughput** strategy does this; the **EP** knob in the Playground mirrors it).

- **Multi-token prediction (MTP):** the **Low-Latency** strategy enables MTP speculative decoding (EAGLE: `--speculative-algorithm EAGLE --speculative-num-steps 3 --speculative-eagle-topk 1 --speculative-num-draft-tokens 4`). Toggle it independently via the **Speculative Decoding** card in the Playground.

- **KV cache dtype:** the default keeps full-precision KV (no flag). To trade accuracy for memory, pick `fp8_e4m3` (or `bf16`) in the **KV Cache DType** card of the [Playground](#playground), which appends `--kv-cache-dtype <value>`. `fp8_e4m3` is accuracy-degrading, so it is offered there rather than baked into the verified recipes.

- **Reasoning / tool-call parsers:** append `--reasoning-parser nemotron_3` for structured reasoning traces (the `reasoning_content` field) and `--tool-call-parser qwen3_coder` for tool calling. Both are exposed as toggles in the **Parsers** card of the [Playground](#playground) (§3.1, §3.2).

### 3.2 Reasoning

The model supports two modes — Reasoning ON (default) vs OFF. Launch with `--reasoning-parser nemotron_3` (or toggle **Reasoning Parser** in the **Parsers** card of the [Playground above](#playground)) so the thinking trace lands in `message.reasoning_content`, separate from the answer in `message.content`. Reasoning can be turned off per request by setting `enable_thinking` to `False`.

<Accordion title="Reasoning Example (Python)">

```python Example
from openai import OpenAI

client = OpenAI(
    base_url="http://localhost:30000/v1",
    api_key="EMPTY",
)

# Reasoning on (default)
print("Reasoning on")
resp = client.chat.completions.create(
    model="nvidia/NVIDIA-Nemotron-3-Ultra-550B-A55B-BF16",
    messages=[
        {"role": "system", "content": "You are a helpful assistant."},
        {"role": "user", "content": "Plan a 3-step approach to debug a flaky integration test. Keep the thinking process short."}
    ],
    temperature=1,
    max_tokens=1024,
)
print(f"Reasoning: \n{resp.choices[0].message.reasoning_content[:200]}... \nContent: \n{resp.choices[0].message.content[:200]}...")
print("\n")
# Reasoning off
print("Reasoning off")
resp = client.chat.completions.create(
    model="nvidia/NVIDIA-Nemotron-3-Ultra-550B-A55B-BF16",
    messages=[
        {"role": "system", "content": "You are a helpful assistant."},
        {"role": "user", "content": "Give me 3 facts about SGLang."}
    ],
    temperature=0,
    max_tokens=256,
    extra_body={"chat_template_kwargs": {"enable_thinking": False}}
)
print(f"Content: \n{resp.choices[0].message.content[:200]}...")
```

### 3.4 Controlling Reasoning Budget

The `reasoning_budget` parameter limits the length of the model's reasoning trace. When the reasoning output reaches the specified token budget, the model attempts to gracefully end the reasoning at the next newline. If no newline is encountered within 500 tokens after reaching the budget threshold, the reasoning trace is forcibly terminated at `reasoning_budget + 500` tokens.

<Accordion title="Reasoning Budget Client (Python)">

```python Example
from typing import Any, Dict, List
import openai
from transformers import AutoTokenizer

class ThinkingBudgetClient:
    def __init__(self, base_url: str, api_key: str, tokenizer_name_or_path: str):
        self.base_url = base_url
        self.api_key = api_key
        self.tokenizer = AutoTokenizer.from_pretrained(tokenizer_name_or_path)
        self.client = openai.OpenAI(base_url=self.base_url, api_key=self.api_key)

    def chat_completion(
        self,
        model: str,
        messages: List[Dict[str, Any]],
        reasoning_budget: int = 512,
        max_tokens: int = 1024,
        **kwargs,
    ) -> Dict[str, Any]:
        assert (
            max_tokens > reasoning_budget
        ), f"reasoning_budget must be smaller than max_tokens. Given {max_tokens=} and {reasoning_budget=}"

        # 1. first call chat completion to get reasoning content
        response = self.client.chat.completions.create(
            model=model,
            messages=messages,
            max_tokens=reasoning_budget,
            **kwargs
        )

        reasoning_content = response.choices[0].message.reasoning_content or ""

        if "</think>" not in reasoning_content:
            # reasoning content is too long, closed with a period (.)
            reasoning_content = f"{reasoning_content}.\n</think>\n\n"

        reasoning_tokens_used = len(
            self.tokenizer.encode(reasoning_content, add_special_tokens=False)
        )
        remaining_tokens = max_tokens - reasoning_tokens_used

        assert (
            remaining_tokens > 0
        ), f"remaining tokens must be positive. Given {remaining_tokens=}. Increase max_tokens or lower reasoning_budget."

        # 2. append reasoning content to messages and call completion
        messages.append({"role": "assistant", "content": reasoning_content})
        prompt = self.tokenizer.apply_chat_template(
            messages,
            tokenize=False,
            continue_final_message=True,
        )

        response = self.client.completions.create(
            model=model,
            prompt=prompt,
            max_tokens=remaining_tokens,
            **kwargs
        )

        response_data = {
            "reasoning_content": reasoning_content.strip().strip("</think>").strip(),
            "content": response.choices[0].text,
            "finish_reason": response.choices[0].finish_reason,
        }
        return response_data
```

</Accordion>
