docker run --gpus all \

    -v ~/.cache/huggingface:/root/.cache/huggingface \
    --env "HF_TOKEN=<your-hf-token>" \

Pick your hardware + recipe to generate the launch command. The two serving strategies cover the common operating points:

- **Low-Latency** — fastest reply for a single user. Pick for chat. DP-Attention is off.
- **High-Throughput** — most tokens per second across many concurrent users. Best for batch jobs. Enables data-parallel attention (`--dp N --enable-dp-attention`).

## 1. Model Introduction

## 2. Configuration Tips

- **Speculative decoding (MTP / EAGLE)** can significantly reduce latency for interactive use cases. It is on by default on NVIDIA in both serving strategies above; toggle it in the **Speculative Decoding** card of the [Playground](#playground). EAGLE speculative decoding is **not currently supported on AMD** for GLM-5.1.
- **DP Attention** (the **High-Throughput** strategy) enables data-parallel attention for higher throughput under high concurrency. It trades off low-concurrency latency for high-concurrency throughput — pick **Low-Latency** if your workload is latency-sensitive with few concurrent requests.
- **`--mem-fraction-static`** is recommended for optimal memory utilization; adjust it based on your hardware and workload.
- **H100 / H200**: FP8 is the recommended deployment path (`tp=16` on H100, `tp=8` on H200).
- **B300 / GB300**: NVFP4 is the recommended path — `nvidia/GLM-5.1-NVFP4` with `--quantization modelopt_fp4` (`tp=8` on B300, `tp=4` on GB300). These require the CUDA 13 Docker image variant.
- **AMD GPUs (MI300X / MI325X / MI355X)**: BF16 at `tp=8`. Use `--dsa-prefill-backend tilelang --dsa-decode-backend tilelang` for the DSA attention backend, plus `--chunked-prefill-size 131072` and `--watchdog-timeout 1200` (20 minutes for weight loading). The FP8 checkpoint also runs on AMD at `tp=8` and uses roughly half the memory of BF16 (~89 GB/GPU vs ~175 GB/GPU).
- GLM-5.1 and DeepSeek-V3.2 share the same model structure. For deeper tuning notes (MTP, DSA kernel, Context Parallel, HiSparse, NVFP4, Index Cache), see the [DeepSeek-V3.2 cookbook page](../DeepSeek/DeepSeek-V3_2).
- **IndexCache**: pass `--json-model-override-args '{"index_topk_pattern": "FFSFSSSFSSFFFSSSFFFSFSSSSSSFFSFFSFFSSFFFFFFSFFFFFSFFSSSSSSFSFFFSFSSSFSFFSFFSSS"}'` to enable the [IndexCache](https://github.com/THUDM/IndexCache) method. This can improve serving efficiency with a small accuracy loss — do **not** enable it for rigorous accuracy evaluations.

## 3. Advanced Usage

GLM-5.1 supports **Thinking mode by default**. Enable the `glm45` reasoning parser (toggle **Reasoning Parser** in the **Parsers** card of the [Playground above](#playground)) to separate the thinking process (returned via `reasoning_content`) from the final answer. To disable thinking and use Instruct mode, pass `{"enable_thinking": false}` via `chat_template_kwargs` at request time.

### 3.1 Reasoning

<Accordion title="Thinking Mode — Default (Python)">

```python Example
from openai import OpenAI

client = OpenAI(
    base_url="http://localhost:30000/v1",
    api_key="EMPTY"
)

# Thinking mode is enabled by default, no extra parameters needed
response = client.chat.completions.create(
    model="zai-org/GLM-5.1-FP8",
    messages=[
        {"role": "user", "content": "Solve this problem step by step: What is 15% of 240?"}
    ],
    max_tokens=2048,
    stream=True
)

# Process the stream
has_thinking = False
has_answer = False
thinking_started = False

for chunk in response:
    if chunk.choices and len(chunk.choices) > 0:
        delta = chunk.choices[0].delta

        # Print thinking process
        if hasattr(delta, 'reasoning_content') and delta.reasoning_content:
            if not thinking_started:
                print("=============== Thinking =================", flush=True)
                thinking_started = True
            has_thinking = True
            print(delta.reasoning_content, end="", flush=True)

        # Print answer content
        if delta.content:
            # Close thinking section and add content header
            if has_thinking and not has_answer:
                print("\n=============== Content =================", flush=True)
                has_answer = True
            print(delta.content, end="", flush=True)

print()
```

To disable thinking and get a direct response, pass `{"enable_thinking": false}` via `chat_template_kwargs`:

<Accordion title="Instruct Mode — Thinking Off (Python)">

```python Example
from openai import OpenAI

client = OpenAI(
    base_url="http://localhost:30000/v1",
    api_key="EMPTY"
)

# Disable thinking mode via chat_template_kwargs
response = client.chat.completions.create(
    model="zai-org/GLM-5.1-FP8",
    messages=[
        {"role": "user", "content": "What is 15% of 240?"}
    ],
    extra_body={"chat_template_kwargs": {"enable_thinking": False}},
    max_tokens=2048,
    stream=True
)

# In Instruct mode, the model responds directly without reasoning_content
for chunk in response:
    if chunk.choices and len(chunk.choices) > 0:
        delta = chunk.choices[0].delta
        if delta.content:
            print(delta.content, end="", flush=True)

print()
```

### 3.2 Tool Calling

GLM-5.1 supports tool calling. Enable the `glm47` tool-call parser (toggle **Tool Call Parser** in the **Parsers** card of the [Playground above](#playground)) to surface structured tool calls via `message.tool_calls`. Thinking mode is on by default; to disable it for tool-calling requests, pass `extra_body={"chat_template_kwargs": {"enable_thinking": False}}`.

<Accordion title="Tool Calling — with Thinking (Python)">

```python Example
from openai import OpenAI

client = OpenAI(
    base_url="http://localhost:30000/v1",
    api_key="EMPTY"
)

# Define available tools
tools = [
    {
        "type": "function",
        "function": {
            "name": "get_weather",
            "description": "Get the current weather for a location",
            "parameters": {
                "type": "object",
                "properties": {
                    "location": {
                        "type": "string",
                        "description": "The city name"
                    },
                    "unit": {
                        "type": "string",
                        "enum": ["celsius", "fahrenheit"],
                        "description": "Temperature unit"
                    }
                },
                "required": ["location"]
            }
        }
    }
]

# Make request with streaming to see thinking process
response = client.chat.completions.create(
    model="zai-org/GLM-5.1-FP8",
    messages=[
        {"role": "user", "content": "What's the weather in Beijing?"}
    ],
    tools=tools,
    stream=True
)

# Process streaming response
thinking_started = False
has_thinking = False

for chunk in response:
    if chunk.choices and len(chunk.choices) > 0:
        delta = chunk.choices[0].delta

        # Print thinking process
        if hasattr(delta, 'reasoning_content') and delta.reasoning_content:
            if not thinking_started:
                print("=============== Thinking =================", flush=True)
                thinking_started = True
            has_thinking = True
            print(delta.reasoning_content, end="", flush=True)

        # Print tool calls
        if hasattr(delta, 'tool_calls') and delta.tool_calls:
            # Close thinking section if needed
            if has_thinking and thinking_started:
                print("\n=============== Content =================", flush=True)
                thinking_started = False

            for tool_call in delta.tool_calls:
                if tool_call.function:
                    print(f"Tool Call: {tool_call.function.name}")
                    print(f"   Arguments: {tool_call.function.arguments}")

        # Print content
        if delta.content:
            print(delta.content, end="", flush=True)

print()
```

</Accordion>
