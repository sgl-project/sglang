## 1. Model Introduction

The MiMo-V2.5 family from XiaomiMiMo includes the native multimodal [XiaomiMiMo/MiMo-V2.5](https://huggingface.co/XiaomiMiMo/MiMo-V2.5) checkpoint and the larger text-only [XiaomiMiMo/MiMo-V2.5-Pro](https://huggingface.co/XiaomiMiMo/MiMo-V2.5-Pro) checkpoint. Both use Mixture-of-Experts (MoE) architecture, hybrid attention, long-context serving, and Multi-Token Prediction (MTP).

<Note>
MiMo-V2.5-Pro is text-only in the current SGLang support path. Multimodal support for the Pro variant is planned for a future release.
</Note>

**Key features:**

- **Sparse MoE scale**: Choose the 310B multimodal checkpoint or the 1.02T Pro checkpoint based on workload and cluster capacity.
- **Long context**: Both variants support long-context serving up to 1M tokens.
- **Hybrid attention**: Uses the MiMo-V2 hybrid attention architecture for long-context efficiency.
- **MTP acceleration**: Supports 3-layer MTP through SGLang's EAGLE speculative decoding path.
- **Reasoning and tool use**: Use the MiMo tool-call parser; the Pro recipe uses the `mimo` reasoning parser.

## 2. SGLang Installation

Refer to the [official SGLang installation guide](/docs/get-started/install). Use a recent SGLang build that includes MiMo-V2.5 and MiMo-V2.5-Pro support.

## 3. Model Deployment

Use the configuration selector below to generate the deployment command. The generator mirrors the MiMo-V2.5 launch configurations from the SGLang support PRs and emits `sglang serve` syntax for current SGLang CLI usage.

For MiMo-V2.5-Pro multi-node deployments, set these distributed placeholders before launch:

- `LWS_LEADER_IP`: rendezvous host IP.
- `LWS_WORKER_INDEX`: node rank.
- `LWS_GROUP_SIZE`: total number of nodes.

### Configuration tips

- **Platform**: These recipes target NVIDIA CUDA deployments using FA3 attention and DeepEP.
- **MiMo-V2.5**: The generator emits a multimodal recipe with `pp=1`, `dp=2`, `tp=8`, `--mm-enable-dp-encoder`, `--attention-backend fa3`, and `--mm-attention-backend fa3`.
- **MiMo-V2.5-Pro**: The generator emits a text-only multi-node recipe with `pp=1`, `dp=2`, `ep=16`, `tp=16`, EAGLE speculative decoding, and `--context-length 1048576`.
- **Context length**: Reduce `--context-length` if you need more memory headroom for your workload.
- **DeepEP dispatch buffer**: For the Pro recipe, tune `SGLANG_DEEPEP_NUM_MAX_DISPATCH_TOKENS_PER_RANK` together with `--max-running-requests` and `--speculative-num-draft-tokens` for high-concurrency serving.

## 4. Model Invocation

Once the server is running, send a chat request to the OpenAI-compatible endpoint. Use `mimo-v2.5` for the standard variant and `XiaomiMiMo/MiMo-V2.5-Pro` for the Pro variant unless you override `--served-model-name`.

```bash Command
curl http://localhost:9001/v1/chat/completions \
  -H "Content-Type: application/json" \
  -d '{
    "model": "mimo-v2.5",
    "messages": [
      {"role": "user", "content": "Summarize the main differences between dense and MoE language models."}
    ],
    "max_tokens": 512,
    "temperature": 0.7
  }'
```

For multimodal request formats, use the OpenAI-compatible message content schema described in [Basic API Usage](/docs/basic_usage/send_request).
