# L2 swe/tb2

## Description
Specialist LoRA for coding and terminal benchmark workflows. The slash-separated task label means one adapter covers SWE-Gym/SWE-Bench-style repository repair and Terminal-Bench/TB2-style command-line software engineering tasks. L2 is not for general technical discussion; conceptual explanations, comparisons, report-review checklists, or ML/system tradeoff questions stay on L0 unless the user explicitly asks to inspect code, edit files, run commands, debug a repository, or complete a terminal benchmark.

## Routing Rules
- Choose L2 for prompts that require source-code changes, debugging, repository inspection, tests, stack traces, failing behavior, patches, scripts, shell commands, or terminal workflows.
- Choose L2 for SWE-Gym, SWE-Bench, Harbor, Pi, or Terminal-Bench/TB2 prompts, especially when the prompt asks to inspect a repo, use tools, make a targeted code fix, and run verification.
- Choose L2 for Terminal-Bench tasks that ask the agent to create a service, write scripts, install packages, process files under `/app` or `/data`, produce compiled/programmatic artifacts, or run command-line validation.
- Prefer L2 over L4 when the primary task is repository repair or benchmark coding, even if the prompt mentions a workspace or local files.
- Do not choose L2 for general questions about LoRA, prompt engineering, KV cache, attention, quantization, routing benchmarks, or experiment reports unless the user asks for a code change, repository inspection, terminal command, test run, or implementation patch.
- Do not choose L2 for generic checklists, critiques, study notes, high-level comparisons, naming ideas, or conceptual review guidance.
- Do not choose L2 for consumer-service booking/order workflows, customer-service policies, real-world personal coordination, or general chat.
