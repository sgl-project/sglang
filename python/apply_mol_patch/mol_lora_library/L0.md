# L0 chat

## Description
Entry LoRA and fallback route for general chat. L0 handles open-domain conversation, general reasoning, conceptual explanation, translation, rewriting, summarization, brainstorming, critique/checklists, and ambiguous requests that do not clearly match L1-L4. Technical discussion stays on L0 when the user is asking for concepts, comparisons, definitions, tradeoffs, naming, performance summaries, report reviews, or review guidance rather than asking to inspect a repository, modify code, run commands, build a UI, or produce local artifacts.

## Routing Rules
- Start every request on L0.
- Keep decoding on L0 for ordinary chat, language polishing, translation, conceptual explanation, simple advice, or ambiguity.
- Keep decoding on L0 for general technical concepts and ML/system discussions such as KV cache, attention, quantization, LoRA fine-tuning, prompt engineering, routing benchmarks, or model-evaluation reports when no code/file/terminal execution is requested.
- Keep decoding on L0 for generic checklists, critiques, rubrics, report reviews, concise overviews, benchmark-performance summaries, naming ideas, high-level comparisons, and study-style explanations.
- Keep decoding on L0 when words like concise, report, checklist, performance, guide, guidance, write, or review are used in a general-chat sense and the request does not ask for code, local files, UI artifacts, or terminal execution.
- Route away from L0 only when the request clearly needs one of the specialist task families: L1 personal-agent/customer-service workflows, L2 coding/terminal workflows, L3 A2UI/UI-generation workflows, or L4 OpenClaw/Pinch workspace workflows.
- If multiple specialist routes look equally plausible and no route has a clear task-family signal, choose L0.
