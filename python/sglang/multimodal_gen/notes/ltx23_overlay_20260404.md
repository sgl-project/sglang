## 2026-04-04 Prompt-Stage 对拍更新

- 对拍环境：
  - 机器：H100
  - 容器：`sglang_mick`
  - worktree：`/tmp/sglang_ltx23_validation`
  - 固定 prompt：`A beautiful sunset over the ocean`
  - fixed negative prompt：`" "`
- 先做的第一轮 prompt-stage compare：
  - 官方路径：`PromptEncoder` / `EmbeddingsProcessor.process_hidden_states`
  - SGLang 路径：`_gemma_postprocess_func` + `LTX2TextConnectors`
  - 结果：
    - `attention_mask` 完全一致
    - 但 `video/audio prompt embeddings` 只有高相似，不是精确对齐
    - 说明发散最晚不超过 prompt-stage
- 定位到的真实问题：
  - 官方 `LTX-2.3` 的 `FeatureExtractorV2` 已经不是旧的 `pack_text_embeds` 逻辑
  - 官方 2.3 真实行为是：
    - per-token RMS norm
    - flatten hidden states
    - 对 `video/audio_aggregate_embed` 前分别做 `sqrt(target_dim / 4096)` rescale
  - 我们原来的实现仍然在沿用旧的：
    - masked mean + min/max range norm
    - 缺少 2.3 aggregate embed 前的 rescale
- 已落地修复：
  - `configs/pipeline_configs/ltx_2.py`
    - 当 `caption_proj_before_connector=True` 时，`_gemma_postprocess_func` 改走 2.3 的 per-token RMS norm
  - `runtime/models/adapter/ltx_2_connector.py`
    - 在 `video_aggregate_embed` / `audio_aggregate_embed` 前补官方一致的 rescale
  - 补了最小单测锁住 2.3 prompt pre-processing 逻辑
- 修复后重新 compare 的结果：
  - `video_features` cosine:
    - prompt: `0.9999982`
    - negative prompt: `0.9999956`
  - `audio_features` cosine:
    - prompt: `0.9999979`
    - negative prompt: `0.9999948`
  - 含义：
    - `feature_extractor` 级别已经基本对齐
    - 剩余 prompt-stage 漂移主要落在 `connector transformer` 自身，而不是 Gemma hidden-state postprocess
- 当前精度对齐进度：50%
  - 已完全对齐到：`FeatureExtractorV2` 输出（含 2.3 RMS norm + aggregate embed 前 rescale）
  - 下一步：继续二分 `video/audio connector` 第一层 block，优先看 self-attention / final RMSNorm / FFN 的差异
