## 2026-05-07 fresh main TP 分支
- 基于最新 `origin/main` 的 `9dfb1d2e` 新建 `ltx23-tp-psnr20-main-20260507`，不在 #24313 历史分支上继续叠改。
- 只移植当前 TP/official 对齐所需改动：`c7315a78` opt-in transformer fp8-cast、`dc9d299c` fp8-cast LoRA kernel 修正、`a0ddcd27` 多卡 LTX2 case 改 TP、`f95cc143` LTX2.3 T2V seed=42、`4ba533b7` LTX2.3 TP case 显式 fp8-cast。
- 当前精度对齐百分比：待 H100 TP=2 复测；目标 official PSNR > 20，100% 对应 bit-exact。
- `68690fbb` 在 H100_2 容器 GPU2/3 跑 `ltx_2.3_two_stage_t2v_2gpus`：server 确认 `--tp-size 2`、`--transformer-weight-quantization fp8-cast`、text_encoder `torch_sdpa`。
- 结果：official GT `PSNR=17.3501, sim=0.9787, SSIM=0.5183, MAD=23.5478`，未达 `PSNR>20`；E2E `11128.86ms`，median denoise `245.19ms`，peak `32872MB`。
- 当前精度对齐百分比：按 PSNR 目标折算 `86.8%`（17.3501/20）；比旧分支最新 TP=2 结果一致，说明 fresh main/rebase 不改变核心精度 gap。
- `68690fbb` 追加验证：全局 `--attention-backend torch_sdpa` 反而下降到 `PSNR=16.6392`，说明 transformer 侧强制 SDPA 不是当前 TP=2 official gap 的解法。
- `7925e74d` 对 fp8-cast 的 `RowParallelLinear` 改为 fp32 partial + fp32 all-reduce 后再 cast 回输入 dtype，避免每个 TP rank 先各自 bf16 rounding。
- 结果：official GT `PSNR=18.2666, sim=0.9665, SSIM=0.5972, MAD=20.8859`；E2E `12351.26ms`，median denoise `262.56ms`，peak `32944MB`。
- 当前精度对齐百分比：按 PSNR 目标折算 `91.3%`（18.2666/20）；确认 TP partial rounding 是真实误差源，但单独修复不足，且有约 7-11% 性能代价。
- `7925e74d` 下切换 `SGLANG_USE_SGL_FA3_KERNEL=false` 结果仍为 `PSNR=18.2666`，说明当前差距不是 SGL FA3 wrapper vs external FA3 的实现差异。
- `7925e74d` 单卡 `sglang generate` 对同一 official 2GPU GT：默认 transformer mixed FA/SDPA 路径 `PSNR=18.5239`，精度对齐百分比 `92.6%`。
- `7925e74d` 单卡强制 `transformer=torch_sdpa,text_encoder=torch_sdpa`：`PSNR=18.1441`，精度对齐百分比 `90.7%`；全 SDPA 更差，不能作为对齐方向。
- 结论：TP fp32-reduce 后已经接近单卡上限，剩余 gap 主要是 native 单卡 transformer/pipeline 语义与 official 的差距，不是 TP 或 FA wrapper 单点问题。
- `pending` 代码复核发现 LTX2.3 safetensors metadata 的 transformer config 使用 `frequencies_precision="float64"`，official `model_configurator.py` 会据此启用 double-precision RoPE；SGLang 只读取 `rope_double_precision`/`arch.double_precision_rope`，可能导致 transformer RoPE 频率走 float32 路径。
- 已修改 `ltx_2.py`：当 `frequencies_precision="float64"` 时启用 transformer RoPE double precision。当前精度对齐百分比：待 H100 TP=2 复测，预计影响 single-GPU 与 TP 共同 gap，性能影响应低于 denoise 主计算噪声。
