# KV Transfer 限速口径背景与竞争流量阶段报告

**日期:** 2026-06-24

**目标:** 记录 KV Transfer 在 fixed 限速口径下，不同链路拆分、背景占比和并发 flow 竞争对延迟、公平性和有效带宽的影响；正文只保留已经确认语义正确的 fixed 矩阵结果和 competition 并发结果。

## 当前实验范围

当前保留八类 fixed profile：

```text
200_2x100: 两路拆分，目标口径 cap100
200_4x50: 四路拆分，目标口径 cap50
400_2x200: 两路拆分，目标口径 cap200
400_4x100: 四路拆分，目标口径 cap100
200_1x200_head_rdma: 单路 head RDMA，目标口径 cap200
200_1x200_head_tcp: 单路 head TCP，目标口径 cap200
200_1x200_tail_single_nic: 单路 tail single NIC，目标口径 cap200
800_4x200: 四路拆分，目标口径 cap200
```

每个 profile 均保留四档背景占比：

```text
bg1 / bg10 / bg50 / bg90
```

fixed profile 中保留的结果均为完整 21 行汇总表，全部 `error_count=0`。

另追加一组 competition 并发竞争实验：

```text
competition fair: 多个前景 KV transfer 同时启动，每个 flow 按 200/N Gbps 做 per-flow cap
competition uncapped: 多个前景 KV transfer 同时启动，不加 per-flow cap，作为自然竞争对照
```

fixed 结果均为完整 21 行汇总表且 `error_count=0`；competition 结果均为 per-flow 单次并发传输汇总且 `ret=0`。

## 观察

1. `200_2x100 cap100` 与 `200_4x50 cap50` 的 fixed 结果非常接近：大包段随 bg1、bg10、bg50、bg90 约为 23.0、20.9、11.6、2.33GiB/s。
2. `400_2x200 cap200` 在 bg1/bg10 下高于 200G 组，但低于按理论剩余带宽粗算的上限；bg50/bg90 下随背景占比明显下降。
3. `400_4x100 cap100` 的比例关系最清楚：大包段约 bg1 46.0 -> bg10 41.8 -> bg50 23.3 -> bg90 4.65GiB/s。
4. 新增 `200_1x200_head_rdma cap200` 与原 200G RDMA 类结果贴近：bg1/bg10/bg50/bg90 大包段约为 23.03、20.94、11.64、2.33GiB/s，符合单路 cap200 扣除背景后的剩余带宽方向。
5. 新增 `200_1x200_head_tcp cap200` 是明显异常/瓶颈组：bg1 大包段约 0.39GiB/s，bg10/bg50/bg90 约 0.252GiB/s，未跟随 cap200 剩余带宽比例变化。
6. 新增 `200_1x200_tail_single_nic cap200` 随背景占比增加单调下降，但低背景档低于 head RDMA：bg1/bg10/bg50/bg90 大包段约为 17.7、16.8、10.9、2.33GiB/s。
7. 新增 `800_4x200 cap200` 随背景占比增加下降：大包段约 bg1 61.5 -> bg10 54.8 -> bg50 32.7 -> bg90 9.31GiB/s；bg90 接近 4 lanes * 20Gbps，低背景档仍存在聚合上限。
8. competition fair 组验证了 per-flow cap 的公平性：1/2/4/8 个 2GiB flow 分别稳定在约 23.25、11.63、5.82、2.91GiB/s/flow，latency 也按 flow 数近似线性上升。
9. competition uncapped 组显示自然竞争不稳定：2 flows equal 时两个 flow 都约 33GiB/s，但 4/8 flows equal 时出现明显快慢分层，快 flow 可到 32-35GiB/s，慢 flow 只有约 11-14GiB/s。
10. 2 flows 混合大小时，fair 组两个 flow 都稳定在约 11.6GiB/s；uncapped 组不按大小公平分配，短 flow 在 256MiB/512MiB/1GiB 档只拿到约 14.25/17.44/17.69GiB/s。
11. 当前 32 组 fixed 汇总表均为完整 21 行且 `error_count=0`；18 组 competition case 均已记录 per-flow 汇总且 `ret=0`。

## 简短结论

本报告当前保留 32 组 fixed 有效结果和 18 组 competition 并发竞争结果。原有 `200_2x100 cap100`、`200_4x50 cap50`、`400_2x200 cap200`、`400_4x100 cap100` 四个 fixed 矩阵已经完成；2026-06-24 追加的 `200_1x200_head_rdma cap200`、`200_1x200_head_tcp cap200`、`200_1x200_tail_single_nic cap200`、`800_4x200 cap200` 四个 fixed 矩阵也已完成。除 head TCP 外，fixed 大包段整体与“背景占用后前景拿剩余带宽”的方向一致。

新增结果里，`200_1x200_head_rdma cap200` 最贴近单路 cap200 的剩余带宽预期；`200_1x200_tail_single_nic cap200` 在低背景档存在 single NIC/tail path 上限；`800_4x200 cap200` 在低背景档存在聚合上限；`200_1x200_head_tcp cap200` 的吞吐主要受 TCP path 或实现侧瓶颈限制，不能按背景占比模型解释。

competition 结果里，fair cap 能把同一条 RDMA path 上的并发 flow 稳定切成可预期份额；uncapped 虽然单个或少量 flow 的吞吐更高，但 4/8 flow 并发时会出现明显不公平，说明如果目标是可预测 tail latency 和跨 flow 公平性，per-flow cap 是必要控制项。
